这本书是为一类特定读者写的：你有扎实的RL数学底子，上过 GAMES103 那样的物理仿真课，但 VLA 对你还是新世界。这一节给你一张路线图，外加两座专门为你搭的桥——让你每遇到一个陌生概念，都能踩着已知的东西走过去。

全书结构：Part F（基础） + 10 个 Part + 附录

如果你发现自己对现代 AI 的基本概念还不熟悉（Transformer、Attention、Token、Cross Entropy、Gym、概率分布等），建议从 Part F · Foundations 开始。它专门为“有 RL + GAMES103 背景但缺少现代大模型基础”的读者准备的补完章节。

两座桥的约定

这本书最核心的教学手法，是两座桥。每当引入一个 VLA 新概念，我们会尽量从你已经掌握的东西搭一座桥过去——一边是 RL，一边是 GAMES103。桥会长这样：

从 RL 看 这座桥连接你的 RL 直觉

带这个标记的盒子，会把眼前的新概念翻译成你熟悉的 RL 语言：policy、value、reward、policy gradient、TD、actor、stochastic policy……举个例子，后面讲生成式策略时，我们会说它「最接近一个 stochastic actor，但用 behavior cloning 而非 policy gradient 训练」。

从 GAMES103 看 这座桥连接你的仿真直觉

带这个标记的盒子，会借用你在 GAMES103 学的物理仿真知识：刚体、约束、时间步进、数值积分（forward Euler）、向量场……例如讲 flow matching 推理时，我们会说「解 ODE 的 forward Euler 积分，正是你学过的那套数值积分；rectified flow 的直线路径就像积分一条几乎匀速的轨迹，步长少也准」。Isaac Sim 的物理循环，本质就是你学过的刚体步进被 GPU 并行到上千个 env。

不是每个概念都两座桥都搭——哪座更顺手就用哪座。遇到桥，请停一下，先在脑子里把新旧概念对齐，再往下读。这是这本书希望你养成的读法。

英文术语的约定

本书保留所有英文术语——架构名、命令、性能指标、概念名一律不翻译，用 这种高亮 标出。原因有二：一是这些词正是你将来读 paper、读 issue、和同行交流时要用的；二是中文译名往往多套并存，反而增加歧义。所以你会看到 flow matching、action chunk、covariate shift、EvalClient 这样的词直接出现在中文句子里——这是刻意的。偶尔我们用 斜体高亮 来强调一个需要你特别留意的词。

一个会反复出现的口径提醒

EBench 有两套口径：README / paper 用愿景语言（如「5-axis atomic diagnostic」「794 task instances」「Test-Mini」），线上 docs 给可验证事实（「3 tracks × 3 splits、26 tasks、6,600 episodes」，且 split 细分标 WIP）。本书会明确区分二者：讲愿景时标注「README/paper 愿景」，讲机制时用 docs 与代码事实，绝不把两者混为一谈。
