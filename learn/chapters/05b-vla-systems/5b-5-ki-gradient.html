Knowledge insulation：梯度到底怎么走

5.8 用「乱流冲击稳定系统」讲过 knowledge insulation 的为什么。但只有直觉容易留下两个悬而未决的疑问：「KI 是不是就是冻结 backbone？」「action expert 是不是看不到 VLM 了？」这一节拿显微镜对准计算图——把前向和反向分开画，逐条 loss 追踪梯度落点。看完你能一句话答对所有 KI 相关的面试题。

先复述一遍病灶：后训练为什么会「污染」

VLM backbone 在预训练里攒下的资产是一个组织良好的语义空间：物体识别、空间关系、常识、图文对齐。后训练接上的 action expert 却是随机初始化的——它初期的预测是纯噪声，flow matching loss 于是巨大，反向传播的梯度既大又没方向感。如果放任这股梯度一路回传，backbone 的权重会被迫「为了拟合眼前这批机器人动作」而移动，把攒下的语义空间拉歪——这就是后训练污染（KI 论文实测：知识迁移变差、训练变慢）。

问题的精确表述是：我们想让 expert 用 backbone 的表示，但不想让 expert 的误差改写 backbone。「用」发生在前向，「改写」发生在反向——所以解法必须区分前向和反向。

前向一张图，反向另一张图

这是本节的核心心法：计算图有两副面孔。前向传播决定「谁能读到谁」，反向传播决定「谁能改写谁」——两者可以不对称，而 stop-gradient 正是制造这种不对称的工具。KI 的完整规则只有三条：

  • 前向：image/text/state → VLM backbone → hidden prefix → action expert → 连续 chunk。expert 完整读取 backbone 的 hidden states，KI 开不开都一样。
  • 反向（CE 类）：VQA / language / annotation CE 与 FAST action-token CE 的梯度照常更新 backbone——这是干净、稳定的监督。
  • 反向（flow）：flow matching loss 的梯度更新 action expert 和 projection，但在流向 backbone 的路口被 stop-gradient 截断。

KI ≠ 冻结：一张误解对照表

widget 里最该看清的一点：KI 开着时，backbone 依然在被更新——被 CE 类监督温和地更新。「隔离」隔的是某一条梯度通路，不是所有通路。把常见误解排成一张表：

误解正确说法
KI = 冻结 backbone不对。backbone 仍被 VQA / language / FAST / annotation 的 CE loss 更新；KI 只阻断 flow loss 这一条回流。冻结是「谁都不许改」，KI 是「只许干净监督改」。
KI 是推理技巧不对。KI 是训练时机制——推理没有梯度，无所谓隔不隔离。
action expert 看不到 VLM不对。前向完整可读（expert 全靠 hidden prefix 感知世界）；被切断的只是反向。
连续动作训练一定比离散强不一定。连续适合实时控制，但随机初始化 + 大梯度使它训练不稳——所以才需要 FAST（先离散教会 backbone）与 KI（隔离乱梯度）保驾。
后训练必然破坏 backbone不必然，但没有隔离、数据又窄时风险显著升高——KI 论文的对照实验正是证据。
从 RL 看 你早就用过 stop-gradient：semi-gradient TD 与 target network

「故意不让某条梯度流回去」在 RL 里是老朋友。算 TD 目标 $r + \gamma V_\phi(s')$ 时，你不对目标项求导（semi-gradient）——否则价值函数会追着自己跑、训练发散；DQN 的 target network 更进一步，把目标参数整个隔离冻结。KI 的结构一模一样：识别出一条会造成不稳定的梯度通路，手术式切断，其余照常。区别只在动机——TD 是防自举发散，KI 是防新模块的噪声梯度污染预训练资产。

从 LabVLA 看 KI 的工程落地

LabVLA 的 KI posttraining 阶段（80k steps）给这套抽象一个可核对的实例：flow matching loss 与 VLM hidden states 之间放 stop-gradient，DiT expert 与 projection 接住全部 flow 梯度；与此同时 LabEmbodied-Data 的结构化标注（subtask、object-state 等）继续走 annotation CE 更新 backbone——所以这一阶段 backbone 学到的是「实验室语义」，而不是「实验室动作的噪声」。两条监督在同一个 batch 里并行，各回各家。

对照阅读：openpi 的现状

再提醒一次 5.8 末尾那个 caveat：开源 openpi 目前只带 flow matching head，KI 论文里「离散 FAST 监督 backbone + stop-gradient 隔离连续头」的完整双路训练并非开箱即用。读 6.1 的代码时别找错东西。

这一节你需要带走的 · 下一节

(1) 计算图有两副面孔：前向决定谁读谁，反向决定谁改谁，stop-gradient 让两者不对称；(2) KI 三条规则——expert 前向可读 hidden states；CE 类梯度照常更新 backbone；flow 梯度只更新 expert / projection，回 backbone 的路被截断；(3) KI ≠ 冻结——backbone 仍被干净监督更新；它和 semi-gradient TD / target network 是同一种手术。
