多源数据怎么混：有什么监督训什么

上一节账本的「输入」栏里挤着一堆名字：Robointer-VQA、DROID、AgiBot、OXE-AugE、LabEmbodied-Data、web 图文……它们不是同一种东西。有的有动作标签，有的只有文字答案；有的教「看懂」，有的教「怎么动」。这一节回答一个朴素但常被想错的问题：格式各异的多源数据，怎么混进同一个模型的训练里？答案是一条原则：有什么监督，就训什么 loss——绝不硬凑。

又一个要拆的误区

误区

「VLA 的训练数据都是『成功画面 + 轨迹』。」——不对。大部分预训练数据里根本没有机器人动作。VQA 数据是「图 + 问题 → 答案」，web caption 是「图 → 描述」，它们一个动作标签都没有，却是 VLA 泛化能力的重要来源。

把 5b-3 账本里出现过的数据全部摊开，按「样本长什么样 / 训练什么能力 / 走什么 loss」分类：

数据类型例子样本长什么样训练什么loss
VQA / 语义理解Robointer-VQA、web VQAimage + question → answer看懂物体、关系、任务阶段CE
高层子任务文本π0.5 的 subtask 数据observation + task → subtask 文本先想「接下来做什么」CE

注意最关键的分界线：有没有 action 标签。前两行和最后一行没有——它们只能（也只应该）训练 backbone 的语义能力；中间两行有——只有它们能监督 action token / action expert。同一批轨迹数据还会在不同阶段换 loss：预训练走 FAST 离散 CE（喂 backbone），后训练走连续 flow（喂 expert）——这正是 5b-3 两阶段的数据视角。点下面的图亲自走一遍每条通路：

关键原则：没有监督的字段，不要伪造监督

把异构数据塞进同一个 batch，最自然的错误做法是「格式对齐」：所有样本都凑齐所有字段，VQA 样本没有动作？填全零动作。这是数据投毒——模型会认真学习「看到这类图就输出零动作」。正确做法是三件套：

多源混训三件套

① 统一 batch schema：所有样本共用一个超集结构（image、text、state、action chunk、annotation……）；② mask 标记有效字段：每个样本带一份「哪些字段真实存在」的掩码；③ 有什么监督算什么 loss：action mask 为假的样本，flow / FAST loss 直接跳过，只算它有的 CE loss。

用伪代码写出来不过几行，但它是整个多源混训的承重墙：

multi_source_loss（示意伪代码）
loss = 0
if batch.has_text_target:          # VQA / caption / subtask / annotation
    loss += ce_loss(vlm_logits, batch.text_target)
if batch.has_action_target:
    if stage == "pretrain":        # 离散：FAST token 走语言那套
        loss += ce_loss(vlm_logits, fast_tokenize(batch.action_chunk))
    else:                          # 连续：flow matching 只喂 expert
        loss += flow_mse(expert_pred, batch.action_chunk,
                         dim_mask=batch.active_action_dims)  # 只在活跃维平均！

最后一行的 dim_mask 呼应 5b-3 结尾那个 LabVLA 踩坑实录：mask 原则不仅作用在「样本有没有动作」的粗粒度上，还要下探到「动作向量的哪些维是真的」——32 维 padding 里只有 8 维活跃时，在错误的分母上平均会把梯度悄悄缩小 4 倍。

从 RL 看 这就像 off-policy 数据的重要性修正——只是这里修正的是「监督的存在性」

RL 里混不同来源的经验时，你会用 importance weight 修正分布差异；这里混不同来源的监督时，用 mask 修正「标签存在性」差异。共同点是：混合本身不难，难的是别让不该贡献梯度的东西贡献梯度。全零动作之于 VLA，就像把随机策略的轨迹当 on-policy 数据直接做 policy gradient——形式上能算，语义上是毒。

以 LabVLA 为例走一遍

LabVLA 的数据配置是这套原则的干净样本。预训练（100k steps）四个源：Robointer-VQA 提供语义问答（无动作，只训 CE）；AgiBot World Beta、OXE-AugE（LeRobot 子集 ≈572k trajectories）、DROID 提供真实轨迹（动作过 FAST tokenizer 走 CE，另有 language subtask 标注也走 CE）。后训练（80k steps）换成 OXE-AugE + LabEmbodied-Data：全部是有动作的轨迹，走连续 flow；其中 LabEmbodied-Data 是仿真合成的实验室 rollout，带 subtask、object-state 等结构化标注——标注继续用 CE 喂 backbone（这是 KI 之下 backbone 仍在更新的原因，5b-5 展开）。

值得停一秒的对照：EBench-Dataset 也是「rule-based 合成 + 人类 teleop」的混合（2.4），文档里那句「teleop 抖动会被学进去；要平滑可调高 rule-based 子集权重」——正是混合权重这个旋钮在起作用。多源混训的第二个自由度（除了 mask）就是各源的采样比例，它直接塑造模型的行为风格。

这一节你需要带走的 · 下一节

(1) 多源数据的分界线是有没有 action 标签：VQA / web / 子任务文本只训 backbone 语义（CE），轨迹类才能训动作（FAST CE 或 flow）；(2) 混训三件套 = 统一 schema + mask 有效字段 + 有什么监督训什么 loss，绝不伪造监督（全零动作是毒）；(3) mask 要细到动作向量的活跃维；混合权重是塑造行为风格的旋钮。
