追踪一个 episode 的全程

全书到这里，每个零件你都见过了。最后一节，我们不引入任何新概念——只做一件事：牵着一个 episode，从它睁眼（reset）走到闭眼（打分），看七个 Part 如何咬合成一台完整运转的机器。每经过一个环节，我们就回链到讲它的那一章。这是这本书的收束。

出场角色

先点名。一个 episode 里同时在跑的，是三方：

  • server——云端/本机的 ray_eval_server.py，里面是 Isaac Sim 物理世界（2.1）和 GenManip 任务（2.3）。它是黑盒（4.1）。
  • client——EvalClient，装在你模型的 env 里，负责通信与契约（4.2）。
  • model——你在 7.1 写的 ModelClient，里面是一个 flow-matching VLA（5.2）。

下面这段循环就是它们的全部对话，我们将逐行展开它代表的物理与计算：

一个 episode 的骨架（来自 4.2）
obs = client.reset()                       # ① server 重置世界，吐首帧
done = False
while not done:
    if obs["0"]["obs"]["reset"]:           # ② 新 episode 首步：清模型状态
        model.reset()
    action = model.get_action(obs)         # ③④⑤ 解包→采样→重排成契约
    obs, done = client.step(action)        # ⑥⑦ server 物理步进→新 obs/done
client.close()
# 全部 episode 跑完 → 子目标打分 → 聚合成 capability profile  ⑧⑨

① reset：世界睁眼，吐出第一帧 obs

调用 client.reset()，server 在 Isaac Sim 里把场景摆回初始：物体归位、机器人 lift2（1.3）回到起始位姿，并解析一个任务指令（如"把咖啡豆收进容器"）。它渲染 4 路 480×640 相机（top/left/right/overlook），读出关节/夹爪/底盘 state，打包成 observation 回给 client。这一帧的 reset 字段是 True。

从 GAMES103 看 初始条件

reset 就是给你的物理仿真设初始状态 $x_0$：位置、姿态、速度全部摆好。之后的每一步都是从这个 $x_0$ 出发的数值积分——这正是你在 GAMES103 里对一套刚体/约束系统做的事，只是这里由 PhysX 在 GPU 上跑（2.1）。

② model.reset：清掉上一局的残留

client 看到 obs["0"]["obs"]["reset"] == True，调你的 model.reset()。如果你的策略带 recurrent state、action 队列或历史帧缓存（InternVLA-A1 的 action deque + temporal image pair 就是，6.3），全在这里清空。7.1 把这列为三大坑之一——不清，新 episode 开头会被旧状态污染。

③ 解包 obs：从字典到张量

进入 get_action(obs)，第一步是解包。按 worker_id 取出 obs["0"]["obs"]，里面是 instruction(str)、state.joints(12)、state.gripper(4)、state.base(3)、state.ee_pose、四路 video.{cam}_view(480,640,3 uint8)、timestep、reset（schema 全貌见 4.2）。图像是 uint8 HWC，你得按训练时的口径归一化、相机命名对齐（7.1 第一坑）。低维 state 也要用训练统计量做归一化。

④ flow matching 采样：一团噪声变成一段动作

这是 episode 里最"聪明"的一步，也是 Part 5 的核心。模型不是回归出一个动作均值，而是从分布 $p(A_t\mid o_t)$ 里采样一整段 action chunk $A_t=[a_t,\dots,a_{t+H-1}]$，让它能表达"多种正确做法"的多模态性（5.2）。

具体地：先抽噪声 $A_t^0\sim\mathcal N(0,I)$，然后沿 flow 时间 $\tau\in[0,1]$ 解一个 ODE，用 forward Euler 一步步把噪声"推"成干净动作：

$$A_t^{\tau+\delta}=A_t^{\tau}+\delta\,v_\theta(A_t^{\tau},o_t)$$

其中 $v_\theta$ 是网络学到的向量场。因为 π0 用 rectified flow，路径近乎直线，10 步（$\delta=0.1$）就够（5.2）。观测前缀 $o_t$ 的 KV cache 只算一次，10 步只重算 action token——这套"image/text 走一组权重、state/action 走 action-expert 权重、共享一次 attention"的结构就是 5.3 的 Transfusion/MoE。

从 GAMES103 看 又一次数值积分

采样动作和物理步进，本质是同一件事的两次出现：都是 forward Euler 解 ODE。第④步在 flow 时间 $\tau$ 上积分一个学出来的向量场 $v_\theta$ 得到动作；第⑥步将在物理时间上积分牛顿动力学得到下一个世界状态。你 GAMES103 的积分器直觉，在这条管线里用了两遍。

⑤ 重排成契约：从模型输出到 action dict

模型吐出的 chunk 不一定就长得像 EBench 要的样子。这一步把它重排成契约 dict（4.3）：取需要执行的那几步（receding horizon，5.1——预测 50、只执行 16/25 再 replan），把关节顺序排成 [6 左关节, 2 左夹, 6 右关节, 2 右夹]，底盘三维填进 base_motion，并标好 control_type / is_rel / base_is_rel。三个 baseline 的差异几乎全集中在这一步：π0 的 base_motion = action[16:19] - last 且 base_is_rel=True（6.1）；X-VLA 反缩放 proprio、gripper 阈值门控、base_is_rel=False 独一份绝对底盘（6.2）；InternVLA-A1 的 delta 还原 + 队列 popleft（6.3）。契约相同，填法各异（6.4）。

⑥ client.step → server 物理步进

client 把这个 action dict 发给 server。server 在 Isaac Sim 里执行：PhysX 把关节目标/底盘增量变成力与接触，推进物理时间——刚体碰撞、夹爪闭合摩擦、底盘位移，全是 2.1 那套 USD→PhysX→step→写回 USD 的循环。在 chunk 模式下（--no_save_process，4.4），server 内部整段执行多步，只在 re-inference 点回观测，省掉网络往返。

⑦ 新 obs / done：世界回话

步进后，RTX renderer 重渲 4 路相机、读新 state，打包成下一帧 obs 回给 client，并带一个 done 标志。循环回到第③步——除非 done 为真（任务完成、失败、或步数耗尽），此时这个 episode 结束。整个③→⑦就是 4.2 主循环的一圈。

从 RL 看 这就是 environment step

③–⑦合起来正是你熟悉的 $a_t=\pi(o_t)$、$o_{t+1},\,done=\mathrm{env.step}(a_t)$ 交互回路。唯一不同：env 是个跑在远端的 Isaac Sim 黑盒，$\pi$ 是 flow-matching 采样而非 $\arg\max_a Q$，而且 reward 不是逐步给的——它留到 episode 结束统一结算（下一步）。

⑧ 子目标打分：把一段轨迹压成 [0,1]

episode 收尾，server 按任务的打分规则给一个 task score ∈ [0,1]（3.3）。多数任务不是非 0 即 1：满足某个子目标得一份部分分，子目标求和；并且常带时序条件——前置子目标满足后才评后续（比如"先开微波炉"才轮到评"放进蛋挞"）。这让长流程任务的分数能反映"做到第几步"，而不是一刀切。

⑨ 聚合成 capability profile：回到出发点

很多 episode 这样跑完，每个 task/split 的 episode 分数取平均得 track score，leaderboard 再聚合三个 track（3.2）。在 README 与线上平台上，这些数进一步沿多个能力轴摊开，画成一张 capability profile 雷达图（愿景层，3.5）。

而这，正是我们在 0.1 开篇立下的那个目标：从一个 success rate，到一张多轴能力画像。一个 episode 的全程，最终汇成这张画像里的一笔。

一条线，九个环

reset → 清状态 → 解包 obs → flow-matching 采样 chunk → 重排成契约 → step → 物理步进 → 新 obs/done → 子目标打分 → 聚合成 profile。九个环，每一个都对应这本书的某一章。你现在能从任意一环往两头追到底——这就是读完这本书的意义。
