26 个任务与打分规则

上一节切好了「在哪个分布上测」。这一节钻进 track 内部：26 个任务到底是什么、分成哪三个 family，以及最关键的——一次 episode 的分数 ∈ [0,1] 究竟怎么算出来。打分规则不是细节，它定义了 capability profile 每一根轴的刻度。

三个 family：26 个任务的能力光谱

docs 把 26 个任务按所考能力归入三个 family。它们正好对应开篇那条 long-horizon / dexterous / pick-place 的范式光谱：

  • Long-Horizon（9 个）——多步、有先后依赖的长流程：Bottle / Detergent / Dish / Dishwasher / Fruit / Make Sandwich / Microwave / Pen / Shop。考的是「能不能把一长串子目标按顺序做完」。
  • Pick-and-Place（10 个）——抓取与放置：Utensils to Holder / Bookmark on Book / Soap to Dish / Apple to Fruit Bowl / Remote to Holder / Perfume to Cosmetics Rack / Salt to Spice Rack / Apple from Shelf / Teacup to Saucer / Bowl to Plate。考的是基础原子技能与空间对位。
  • Dexterous & Precise（7 个）——灵巧与精密：Collect Coffee Beans / Flip Cup & Collect Cookies / Frame Against Pen Holder / Install Gear / Peg in Hole / Put Glass in Glassbox / Tighten Nut。考的是毫米级公差的精细操作。

9 + 10 + 7 = 26，正好。下面这个 widget 把全部 26 个任务、它们的 family 归属与打分结构都摊开了，建议边读边点。

按 family 浏览全部 26 个任务；点开任务看它的子目标怎么拆、各占多少分。注意 Long-Horizon 任务的子目标往往带先后顺序。

打分机制：[0,1]、部分分、时序条件

很多 benchmark 的打分是二元的——成了给 1，没成给 0。EBench 的打分要细腻得多，这正是为了支撑「诊断」而非「排名」。一次 episode 的 task score 落在连续区间 [0,1]，由三条规则共同决定：

  • [0,1] 连续：满足整个 goal 给满分；什么都没做成给 0；做了一半，给一个中间值。
  • 部分分（partial credit）：任务被拆成若干子目标，每个子目标分配一份权重，最终分数是已完成子目标权重之和。这样「开了门但没放东西」与「碰都没碰」会得到不同的分。
  • 时序条件（temporal）：有些子目标有先后依赖——后置目标只有在前置目标满足后才被评估。你不能跳过「拧松」直接拿「拧紧」的分。

部分分让分数变成连续信号而不是开关，时序条件保证这个信号诚实——它奖励真实进展，而不奖励钻空子。两者合起来，一个 episode 的分数就能告诉你「模型走到了流程的哪一步才崩」，这正是 capability profile 想要的原料。

两个例子：Microwave 与 Tighten Nut

例一 · Microwave（Long-Horizon，部分分）。这个任务要「开微波炉门 → 放进两个蛋挞 → 关门」。它的子目标权重大致是：开门 1/2，两个蛋挞各 1/6，关门 1/6。

$$\text{score}_{\text{Microwave}} = \tfrac{1}{2}\,[\text{开门}] + \tfrac{1}{6}\,[\text{蛋挞}_1] + \tfrac{1}{6}\,[\text{蛋挞}_2] + \tfrac{1}{6}\,[\text{关门}]$$

方括号是 0/1 指示量。注意权重不均匀：开门占了一半，因为它是后续一切的前提——开不了门，后面全免谈。这就把「难度的结构」编码进了分数。一个只会开门的模型拿 0.5，一个开门 + 放进一个蛋挞的拿 0.5 + 1/6 ≈ 0.67，二者的差恰好量化了「多做成了一步」。

例二 · Tighten Nut（Dexterous & Precise，时序条件）。拧紧螺母这件事天生有顺序：得先把螺母对正、套上，才谈得上拧紧。打分因此带时序门控——「拧紧」这个子目标只有在「对位/套上」已满足时才会被评估。模型若没真正套上就空转手腕，拿不到拧紧的分。这避免了「动作看起来对、但物理上没发生」骗到分数。

从 GAMES103 看 子目标判定来自物理状态，不是动作

这些子目标的「完成」判定，读的是仿真里物体的物理状态——门的铰链角度、蛋挞与微波炉腔体的接触/包含关系、螺母沿螺纹的旋入深度——而不是机器人发出了什么动作。这正是你在刚体/约束仿真里熟悉的那套状态量。temporal 门控本质上是在物理状态机上加了一条「前置态必须先到达」的边，和你写约束求解时的依赖顺序是同一种思路。

这一节你需要带走的

记住：26 个任务 = Long-Horizon 9 + Pick-and-Place 10 + Dexterous & Precise 7；每次 episode 的 task score ∈ [0,1]，由部分分（子目标权重求和）和时序条件（前置满足才评后续）共同算出。Microwave 用不均匀权重把流程结构编进分数；Tighten Nut 用时序门控防止「假动作」骗分。

到此，每个任务内部怎么打分讲清楚了。但 capability profile 还有另一根重要的轴：换了没见过的物体、背景、指令，模型掉了多少分？下一节进入泛化轴——又一段需要分清愿景与现实的内容。
