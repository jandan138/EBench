Capability profile 的愿景

在 0.1 我们说过一句话：不要只看一个 success rate，要看它在各个能力轴上的分解。这一节我们把这个想法讲透——它叫 capability profile，是 EBench 整套方法论的愿景母题。但请先记住一件事：这一节讲的几乎全是「愿景语言」，下一节起我们才落到代码与 docs 的地面真相。

重申：诊断，而不是排名

回顾一下 0.1 的核心不满：success rate 是一个被压扁的标量。模型跑 100 个 episode 成功 63 个，你得到 0.63——然后呢？你不知道它是栽在长流程的中段，还是手不够稳拧不上螺母，还是底盘没走到位，还是换一句指令就听不懂。两个都拿 0.63 的模型可能强在完全不同的地方，而一个标量把这些结构信息全部抹平了。

所以 EBench 想做的不是「再排一次名」，而是诊断：把一次评测的结果，从一个数，摊成一张沿多个能力轴展开的画像。排名是给旁观者看的；画像是给开发者看的——它告诉你「下一步该往哪改」「这个模型会在什么情况下崩」。这是两种根本不同的信息粒度。

从 RL 看 不止看 return，要看 return 的分解

这正是你熟悉的处境。在 RL 里把复杂行为压成单一标量 reward，会丢掉大量结构——reward shaping、multi-objective RL、各种 diagnostic / probing 方法都在试图把丢掉的东西补回来。capability profile 类比过来就是：不要只报 return，要报 return 在各能力轴上的剖面。同样的平均回报，剖面可以截然不同，而剖面才是可操作的。

README/paper 愿景的 5-axis atomic diagnostic

EBench 的 README 和即将发布的 paper，用一套很有野心的语言来描述这张画像：把一个机器人任务拆解成若干原子能力维度（atomic diagnostic axes），让每个维度单独可读。愿景里给出的是 5 个轴：

  • Scene——场景理解与空间布局：物体在哪、桌面/货架/家电怎么排布，机器人能否在杂乱场景里定位目标。
  • Atomic Skill——原子操作技能：抓、放、推、拧、翻转这些最小的动作单元本身做得稳不稳。
  • Horizon——任务时序长度：单步够了，还是要把一长串子目标按先后依赖一步步做完（开门→放进去→关门）。
  • Precision——精度要求：对位公差有多紧，套轴、插孔、拧螺母这种毫米级灵巧度。
  • Mobility——移动性：是否需要先走过去，底盘导航与「移动 + 操作」的协调。

这五个轴的设计动机，是把开篇说的三种范式（long-horizon / dexterous & precise / mobile）背后真正变化的能力因子抽出来，让它们彼此正交、各自可测。一个任务可以同时在 Horizon 与 Precision 上都要求很高（比如装一台多步骤又要精细对位的设备），而 capability profile 想让你看清：模型是栽在「步太多记不住」，还是「手不够准」。

这是「愿景语言」，请标注清楚

上面这套 5-axis atomic diagnostic（Scene·Atomic Skill·Horizon·Precision·Mobility），连同后面会遇到的 4-axis generalization、794 task instances、Test-Mini、Specialist vs Generalist track，都来自 README / paper 的愿景描述。它解释了 EBench「想成为」什么。而当前线上 docs 站能验证的地面真相是另一套口径：3 tracks × 3 splits、26 tasks、数据集 6,600 episodes，且 split 细分仍标 WIP（下一节 3.2 详谈）。本书会一路把愿景和现实分开标注，不让你把「想做的」当成「已实现的」。

换个角度看雷达图：它是诊断仪，不是排行榜

0.1 里我们用雷达图演示过「同样总分、不同画像」。这里我们换一个角度再看一次同一个 widget——不要把它读成「谁的面积大谁赢」。雷达图的价值恰恰不在面积，而在形状：哪条轴塌下去，哪条轴鼓出来。一个面积更小但在你关心的那条轴上更高的模型，对你这次开发可能更有用。

interactivecapability profile：读形状，而不是读面积
勾选不同模型对比。这次请刻意忽略「总面积」，只盯单条轴：如果你正在攻克 Precision，那条轴最高的模型才是你的参照系——哪怕它平均分更低。数值为示意。

这就是「诊断而非排名」最具体的样子：同一张图，排行榜思维只读出一个赢家，诊断思维读出一份改进清单。EBench 的全部方法论，本质上是在为这张图提供可信、可分解的坐标。

这一节你需要带走的

记住三点就够进入下一节：(1) capability profile 的目的是诊断——把一个 success rate 摊成沿能力轴的剖面，服务于「下一步改哪」而非「谁第一」；(2) README/paper 愿景给出 5 个原子能力轴 Scene·Atomic Skill·Horizon·Precision·Mobility，这是愿景语言；(3) 雷达图要读形状不读面积。
