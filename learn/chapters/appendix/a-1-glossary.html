VLA 模型与架构

术语英文全称一句话解释
VLAVision-Language-Action model拿预训练 VLM 骨干，加上输出机器人 action 的能力的模型（1.2）。
VLMVision-Language Model同时吃图像和文本的多模态模型，作 VLA 的骨干。
PaliGemma—π0 用的 3B 开放 VLM 骨干（SigLIP 视觉 + Gemma 语言为背景，paper 只说 3B）。
action chunking—一次预测 H 个未来动作而非每步一个，源自 ACT（5.1）。
action expert—专门处理 state/action token 的一组独立权重（≈300M），与 image/text 权重共享一次 attention（5.7）。
action horizon—一个 chunk 里预测的动作步数 H（π0 H=50）。
receding horizon—开环执行 chunk 的前几步后再 replan，而非执行整段（5.1）。
Mixture-of-TransformersMoT多个专家共享一次 unified self-attention 的结构，InternVLA-A1 用它统一 understanding/generation/action（6.3）。
visual foresight—generation 专家预测未来视觉状态、当作世界模型"想象"的能力（6.3）。
soft prompt—每个数据源/本体一套可学习 embedding，让共享骨干吸收异构性，X-VLA 的核心机制（6.2）。
domain_id—X-VLA 推理时选用哪套 soft-prompt 的索引（一般训练默认 0）。

训练与生成式策略

术语英文全称一句话解释
flow matching—训练一个向量场把噪声推成动作的生成式策略训练法，π0 用条件 flow matching（5.5）。
rectified flow—线性高斯（OT）概率路径下的 flow matching，路径近直线、推理可极少步积分（π0 10 步）。
compounding error—单步误差把机器人推到没见过的状态、误差滚雪球，action chunking 要对抗的问题（5.1）。
covariate shift—策略遇到的状态分布偏离训练分布；compounding error 的根因。
knowledge insulation—训 action expert 时停止其梯度回流 backbone（stop-gradient），保住 VLM 的 web 知识、加快训练（arXiv 2505.23705，5.8、5b-5）。
FAST tokenizerFAST action tokenizer把连续动作离散化的 tokenizer，π0.5 预训练阶段用它做交叉熵 next-token（5.8）。

训练系统与数据生态（Part 5.5）

术语英文全称一句话解释
training exposure—模型考前「见过什么」的总称：是否吃过本 benchmark / 同域数据、同类资产任务等，公平比较的第一变量（5b-8）。
training ledger—本书用的账本读法：每个训练阶段答四问——输入 / 监督 / loss / 更新谁（5b-3）。
pretraining / posttraining / finetuning—VLA 常见三阶段：CE 教 backbone 动作语义 → flow 训 action expert（KI 隔离）→ 对齐具体任务与 schema（5b-3）。
data engine数据引擎自动生成训练数据的系统：造场景→执行→成功过滤→导出；与 benchmark（考场）、recipe（配方）是三个生态位（5b-6）。
LabVLA—实验室特化 VLA：Qwen3-VL-4B backbone + DiT action expert，FAST→flow→KI recipe（arXiv 2606.13578，5b-7）。
RoboGenesis—LabVLA 的 Isaac Sim 数据引擎：资产生成→agentic workflow→成功过滤导出；不需预录轨迹、需可执行专家逻辑（5b-6）。
LabEmbodied-Data—RoboGenesis 导出的实验室演示语料：多相机 + 指令 + state + action + 结构化标注。
LabUtopia—实验室仿真评测套件（LabSim+LabScene+LabBench，30 任务 5 层级），LabVLA 的考场（arXiv 2505.22634）。
AutoBio—生物实验室仿真器 + benchmark：仪器数字化、专用物理插件、透明液体渲染，评现有 VLA（arXiv 2505.14030）。
RDTRobotics Diffusion Transformer1.2B 扩散 Transformer 策略：图像/语言作条件、预测 64 步 chunk——接口层是 VLA、架构层不是「VLM+expert」（arXiv 2410.07864，5b-2）。
DiTDiffusion Transformer扩散/流模型常用的 Transformer 骨架；LabVLA 的 action expert 采用此结构。
OXEOpen X-Embodiment大规模跨本体真实机器人数据集集合；LabVLA 用其增强子集 OXE-AugE（LeRobot 格式 ≈572k 轨迹）。
DROID / AgiBot World—大规模真实机器人操作数据集，VLA 预训练常客（5b-4）。
zero-shot public checkpoint—模型状态五档之一：直接拿公开权重来评，未用本 benchmark 数据后训练（5b-8）。
ID / OODin- / out-of-distribution评测设置：贴近训练分布 vs 故意扰动物体位置/外观/场景以测泛化。

Benchmark、本体与契约

术语英文全称一句话解释
EBenchElemental Mobile Manipulation Benchmark基于 Isaac Sim 的室内 VLA manipulation benchmark（Shanghai AI Lab / InternRobotics）。
lift2—EBench 的本体：dual-arm + 移动底盘 + 4 个 480×640 相机（1.3）。
base_motion—action dict 里的底盘运动 (3,)：Δx, Δy, Δyaw（4.3）。
is_rel—臂动作是否为相对量（相对则 True，否则绝对关节角）。
base_is_rel—底盘是否为相对增量（多数 baseline True；X-VLA 独用 False 绝对底盘）。
EvalClient—genmanip-client 里的客户端类，跑 reset/step/close 主循环（4.2）。
gmp—genmanip-client 的命令行工具：submit/eval/status/plot/clean/visualize（4.5）。
track—评测赛道：mobile_manip / table_top_manip / generalist（3.2）。
split—数据切分：val_train / val_unseen（开放）/ test（留出，驱动 leaderboard）。
capability profile—把单一 success rate 摊成多轴能力画像（雷达图）的 EBench 核心理念（0.1）。
