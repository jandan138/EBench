速查：obs / action / gmp

写 ModelClient 时最常翻回来确认的几样东西，集中在一页：observation schema、action dict、常用 gmp 命令、env 变量。复制即用。完整讲解在 4.2 / 4.3 / 4.5。

Observation schema

按 worker_id 取：obs["<worker_id>"]["obs"]。单 worker 时 worker_id 为 "0"。

obs["0"]["obs"] 的字段
{
  "instruction":      str,                    # 自然语言任务指令
  "state.joints":     (12,),                  # 双臂关节角
  "state.gripper":    (4,),                   # 夹爪
  "state.base":       (3,),                   # x, y, theta（底盘位姿）
  "state.ee_pose":    [[pos, quat], [pos, quat]],  # 左右末端，=14 维
  "video.top_view":     (480, 640, 3),        # uint8, HWC, 0-255
  "video.left_view":    (480, 640, 3),        # uint8
  "video.right_view":   (480, 640, 3),        # uint8
  "video.overlook_view":(480, 640, 3),        # uint8
  "timestep":  int,
  "reset":     bool,    # 新 episode 首步为 True → 调 model.reset()
}
两个必记的坑

(1) 图像是 uint8 HWC 0–255，归一化/转 CHW 要你自己按训练口径做。(2) reset==True 时务必清模型跨步状态（recurrent / action 队列 / 历史帧）。详见 7.1。

Action dict

从 get_action(obs) 返回，键是 worker_id。两种 control_type：

joint_position 模式（最常用）
{
  "0": {
    "action": (16,),          # 6 左关节 + 2 左夹 + 6 右关节 + 2 右夹
    "base_motion": (3,),      # Δx, Δy, Δyaw（底盘运动）
    "control_type": "joint_position",
    "is_rel": False,          # 臂为绝对关节角；相对则 True
    "base_is_rel": True,      # 底盘多为相对增量（X-VLA 独用 False = 绝对）
  }
}
ee_pose 模式（可选）
{
  "0": {
    "action": [pos, quat, gripper],   # 改用末端位姿 + 夹爪
    "base_motion": (3,),
    "control_type": "ee_pose",
    "is_rel": False,
    "base_is_rel": True,
  }
}

注：契约里"动作"19 维 = action(16) + base_motion(3)；generalist 输出 19、fixedbase 16。各 baseline 的通道重排差异见 6.4。

主循环模板

reset / step / close
from genmanip_client.eval_client import EvalClient

client = EvalClient(base_url="http://127.0.0.1:8087",
                    worker_ids=["0"], run_id="my_eval")   # online 再加 token=...
obs = client.reset()
eval_finished = False
while not eval_finished:
    if obs["0"]["obs"]["reset"]:
        model.reset()
    action = model.get_action(obs)
    obs, eval_finished = client.step(action)
client.close()

常用 gmp 命令

server + 本地评测
# 起 server（chunk 模式需 --no_save_process）
python ray_eval_server.py --host 0.0.0.0 --port 8087 --no_save_process

# 提交一个 run（粒度：全部 / 单 track / 单任务）
gmp submit ebench --run_id my_first_run
gmp submit ebench/table_top_manip/val_unseen --run_id my_first_run
gmp submit ebench/table_top_manip/test/collect_coffee_beans --run_id my_first_run

# 跑 client worker（跨机加 --host <ip> --port <port>）
gmp eval -a r5a -g lift2 --worker_ids 0

# 查进度 / 画图 / 清理 / 可视化
gmp status
gmp plot
gmp clean --dry-run          # 或 --all
gmp visualize                # Rerun HTTPS viewer，需 Py3.11+ 和 [visualize] extra
# 结果落在 saved/eval_results/<task>/<run_id>/
online challenge
# 登录 internrobotics.shlab.org.cn/eval 拿 token → $T
gmp online submit \
  --base_url https://internrobotics.shlab.org.cn/eval \
  --token $T --benchmark_set ebench_generalist \
  --model_name my_vla --model_type VLA
# 返回 task_id(=run_id) + endpoint；用 EvalClient(base_url=endpoint, token=$T, run_id=task_id, ...)
# 跑完：
gmp online stop

online 约束：当前仅 ebench_generalist；最多 16 并发 worker；10 分钟无活动断连；同 task_id 可续跑。

gmp 选项速查

选项含义 / 默认
--run_id本次 run 标识；续跑用同一个。
--host / --portserver 地址，默认 127.0.0.1:8087。
--worker_ids本进程的 worker 编号（不同进程不同号）。
--chunk_sizechunk 模式下执行步数（receding horizon，5.1）。
--frame_save_interval每隔多少帧存一次。
