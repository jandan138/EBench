Transformer Block 里到底有什么参数

上一节只讲 Attention 的“谁看谁”。这一节把它放回完整的 Transformer Block：Attention 负责 token 间通信，MLP 负责每个 token 自己加工，LayerNorm 和 Residual 负责让深层网络更好训练。理解这个结构，后面读 action expert、MoE、冻结 backbone 时才知道“到底在动哪些参数”。

Block 的输入输出形状不变

先抓住最重要的稳定点：一个 Transformer Block 吃进去一排向量，吐出来还是同样长度、同样宽度的一排向量。

$$X \in \mathbb{R}^{n \times d} \quad \longrightarrow \quad \text{Block}(X) \in \mathbb{R}^{n \times d}$$

这很关键。因为形状不变，模型可以把很多个 Block 叠起来。第一层可能让“她”看到“小红”，第十层可能已经形成更抽象的“人物关系”，更深层再把语言、图像、状态融合起来。

一层 Transformer 的完整数据流

把 F-1 到 F-2 的内容连起来，一条样本进到某一层 Transformer，大致走下面这条路。这里用文字流程图，是为了让你先抓住数据形状；真实实现会把这些步骤合成高效的矩阵运算。

从字符串到下一层 hidden state
字符串 / 图像 / 状态
  ↓
tokenizer 或模态 encoder
  ↓
token id / patch feature / state vector
  ↓
embedding 查表或线性投影
  ↓
加 position / modality 信息
  ↓
h^0 或当前层输入 H^l                 shape: [n, d]
  ↓
Q = H^l W_Q, K = H^l W_K, V = H^l W_V
  ↓
S = Q K^T / sqrt(d_k)                 shape: [n, n]
  ↓
A = softmax(S + mask)                 shape: [n, n]
  ↓
Z = A V                               shape: [n, d_v]
  ↓
输出投影 + Residual + LayerNorm
  ↓
MLP / FFN + Residual + LayerNorm
  ↓
下一层 hidden state H^{l+1}           shape: [n, d]

这张图回答一个常见困惑：Embedding 只在开头负责把输入变成初始向量；后面每层传递的是 hidden state。Q/K/V 不是新 token，而是当前 hidden state 为了做 Attention 临时投影出来的工作矩阵。

一个 Block 可以拆成四类零件

零件负责什么有没有主要参数新手直觉
Self-Attentiontoken 之间交换信息。有，主要是 $W_Q,W_K,W_V,W_O$。会议室：每个人听别人说话。
MLP / FFN每个 token 独立加工自己的表示。有，通常是两层线性层。私人笔记：每个人整理自己的理解。
LayerNorm把数值尺度调稳。有少量缩放/平移参数。调音台：别让某层声音太大。
Residual保留原输入，加上新变化。没有主要矩阵参数。改稿：不是重写整页，而是在原稿上增量修改。

Attention 参数：四个投影

F-2 里我们见过三套矩阵：$W_Q,W_K,W_V$。完整 Attention 通常还会有一个输出投影 $W_O$，把混合后的信息整理回模型宽度。

$$\begin{aligned} Q &= XW_Q \\ K &= XW_K \\ V &= XW_V \\ A &= \text{Attention}(Q,K,V) \\ \text{Out} &= AW_O \end{aligned}$$

如果忽略多头细节，并假设宽度都是 $d$，这四个矩阵的参数量大约是 $4d^2$。你不需要背这个数，只要理解：Attention 的参数主要在“如何生成 Q/K/V，以及混合后怎么投回去”。

从 GPU 看：没有小人，只有张量

教程喜欢说“每个 token 看谁”，是因为人脑更容易理解。但 GPU 实际保存的是一批矩阵：

张量典型 shape含义
$H$$[B,n,d]$batch 里每个序列位置的当前 hidden state
$Q,K,V$$[B,n,d_k]$由 $H$ 乘不同权重矩阵得到的工作张量
$S=QK^T$$[B,n,n]$所有 query-key pair 的分数表
$A=\text{softmax}(S)$$[B,n,n]$每一行加起来为 1 的 attention weights
$Z=AV$$[B,n,d_v]$所有位置同时完成的 value 加权汇聚

为什么一次矩阵乘法能完成所有 token 的更新？因为 $QK^T$ 会把所有 $q_i\cdot k_j$ 同时算出来，得到整张 $n\times n$ 关系表；接着 $AV$ 又把每一行权重对所有 Value 的加权和同时算出来。拟人化是读者视角，矩阵乘法才是实现视角。

MLP 参数：逐 token 的加工厂

Attention 让 token 互相通信，但通信完还需要加工。MLP 做的就是这件事。它对每个 token 分别应用同一套小网络，不直接让不同 token 交流。

$$\text{MLP}(x)=W_2\,\sigma(W_1x+b_1)+b_2$$

常见设置里，$W_1$ 把维度从 $d$ 放大到 $d_{ff}$，非线性激活后，$W_2$ 再把它压回 $d$。如果 $d_{ff}=4d$，MLP 的参数量大约是 $8d^2$，经常比 Attention 还大。

一个很有用的边界

Attention 负责“谁和谁交换信息”；MLP 负责“每个 token 自己怎么变”。所以后文讲 action expert 时，你会看到一种设计：共享 Attention 让语言/图像/动作通信，但给动作 token 单独的 MLP 或输出头，让动作专门学控制。

Residual 和 LayerNorm：为什么不是可有可无

深层网络最怕两件事：一是每层都把表示改得太猛，原信息丢掉；二是数值尺度乱飘，训练不稳定。Residual 和 LayerNorm 就是为这两个问题服务的。

$$Y = X + \text{Attn}(\text{LN}(X))$$ $$Z = Y + \text{MLP}(\text{LN}(Y))$$

上面是常见的 pre-norm 写法。先 LN，再做 Attention 或 MLP，然后把结果加回原输入。这个“加回去”很重要：Block 学的是增量修改，而不是每层都强迫重建全部信息。

小算例：一个极小 Block 有多少参数

假设模型宽度 $d=4$，MLP 中间维度 $d_{ff}=16$。忽略 bias 和 LayerNorm 的少量参数：

模块近似参数量怎么算
Attention 投影64$4d^2=4\times4\times4$
MLP128$2dd_{ff}=2\times4\times16$
合计约 192这里只是教学玩具，真实模型会大很多。

这个算例不是为了让你会数参数，而是为了建立直觉：模型能力很大一部分来自这些线性投影和 MLP 权重。训练、冻结、LoRA、action expert 讨论的都是“哪些权重要更新，哪些权重要保护”。

从 RL 看 共享大脑与动作小脑

如果把 VLM backbone 看成通用感知和语言大脑，action expert 就像给动作 token 加的专门控制小脑。它通常不会把全部能力从零学一遍，而是在共享表示上增加适合动作预测的参数路径。

三个进阶问题：Multi-Head、KV Cache、VLM/VLA

这些概念后面还会遇到，Part F 先给最小落点。

概念先这样理解为什么重要
Multi-Head Attention不是只做一套 $W_Q,W_K,W_V$，而是并行做多套投影空间。不同 head 可以学不同几何关系：指代、位置、模态对齐、动作依赖等。
KV Cache自回归推理时，过去 token 的 $K,V$ 不变，可以缓存；新 token 只算自己的 $Q$ 去查缓存。生成长序列时避免重复计算过去所有 token 的 K/V。
VLM/VLA 共用 Attention图像、语言、状态进入同一序列后，都是 hidden state 的行；Attention 机制相同。跨模态对齐来自输入投影、位置/模态信息、mask 和训练目标共同塑造。
WQ/WK/WV 的几何意义

它们学到的是“把当前 hidden state 投到适合匹配和取值的空间”。训练后，如果某类 query 应该关注某类 key，loss 会推动对应投影方向靠近；如果不该关注，推动它们远离或让分数被 mask/softmax 压低。所谓 Attention 学高维几何，本质就是这些投影空间和点积关系被训练塑形。

小检查：判断两句话

  1. “MLP 负责让第 3 个 token 直接读取第 10 个 token。”这句话不准确；直接通信主要由 Attention 做。
  2. “Residual 让 Block 可以在原表示上做增量修改。”这句话正确。
常见误解

不要把 Transformer Block 理解成“只有 Attention”。很多参数在 MLP 里；很多训练稳定性来自 LayerNorm 和 Residual。后文如果只盯 Q/K/V，会错过 action expert、梯度隔离、微调账本里的关键。

这一节你需要带走的

一个 Block 的形状是 $n \times d$ 进、$n \times d$ 出。Attention 做 token 间通信，MLP 做逐 token 加工，LayerNorm/Residual 保持训练稳定。下一节 F-4 回到输入端：文字、图像、状态、动作到底怎么变成这排向量。
