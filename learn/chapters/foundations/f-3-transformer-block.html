Transformer Block 从零开始（MLP / Residual / LayerNorm）

现在不再把 Attention、MLP、Residual、LayerNorm 当成四张分散的知识卡。我们只跟一条 hidden state 走一次完整 Block：它先与整句通信，再被加回原表示，再做逐位置计算，再被加回一次。走完以后，再回头拆每个部件为什么存在。

先回到“她 / 小红”的一条 contextual state

从搜索直接跳进来？先拿这张最小地图

一个 Block 的输入是张量 [B,n,d]：B 条序列、每条 n 个 token、每个 token 有 d 个 feature。这里会跟踪“她”这个焦点位置的那一行，但 Attention 每次仍读取整个序列；LayerNorm 和 MLP 才是逐位置处理。每次 + 都是同 shape [B,n,d] 的逐元素相加。

假设前一层已经让“她”的 hidden state 混入了一些“小红”的信息。它不是一个固定意思的词向量，而是这个位置此刻的一排数。一个 Block 的任务不是把这排数推翻重写，而是在保留原表示的前提下，先让它再通信一次，再做一次非线性加工。

先记住无公式的四个角色

Attention通信每个位置可从可见的其它位置取回信息；它消费整个序列。
MLP计算对每个位置当前向量应用同一个非线性函数；它是逐位置的。
Residual保留 + 更新原表示走旁路，分支只需要提供要加上的更新。
LayerNorm稳定每个位置沿自己的 feature 维整理进入计算分支的尺度。

把它们连成一句话：先规范化 → Attention 通信 → 加回原表示 → 再规范化 → MLP 计算 → 再加回。下面先走完这一趟，不在半路停下来讲 Jacobian 或每个矩阵。

一条 hidden state 怎样穿过完整 pre-norm Block

先定义符号，避免公式突然冒出陌生字母：$H^\ell$ 是第 ℓ 层输入的整个 [B,n,d] 张量；$A^\ell$ 是第一次 residual 相加后的临时张量；$LN_1,LN_2$ 是两套独立参数的 LayerNorm；$Attn$ 和 $MLP$ 都输出同样的 [B,n,d] shape。

$$U^\ell=LN_1(H^\ell)$$
$$A^\ell=H^\ell+Attn(U^\ell)$$
$$V^\ell=LN_2(A^\ell),\qquad H^{\ell+1}=A^\ell+MLP(V^\ell)$$

这就是现代常见的 pre-norm 写法。两个加号都合法，是因为 multi-head Attention 经 WO 已回到 d，MLP 经 Linear 2 也回到 d。每一个 + 都沿 B、n、d 对应元素相加，不是把 token 接到序列末尾。

读公式时只跟焦点位置 i
整句 H^ell
  → LN_1
  → Attention(整句)
  → 取回第 i 行 Delta_A
  → H_i^ell + Delta_A
  → LN_2
  → MLP(第 i 行)
  → A_i^ell + Delta_M
  → H_i^(ell+1)

现在跟“她”走六步，不把它误当成孤立向量

  1. 输入是整句 Hℓ；焦点是“她”的那一行。
  2. LN1 分别规范化每个 token 的 feature；“她”只沿自己的 d 个 feature 计算均值和方差。
  3. Attention 让“她”的 Query 与整句可见位置匹配，形成来自其它位置的更新 ΔA。
  4. 第一次 residual 得到 Aℓ = Hℓ + ΔA；原“她”的状态没有被分支直接替换。
  5. LN2 后，MLP 只处理“她”这一行，产生另一个更新 ΔM。
  6. 第二次 residual 得到 Hℓ+1 = Aℓ + ΔM，作为下一层的输入。

Attention 分支只回顾它如何回到宽度 d

F-2.25 已推导 multi-head Attention：每个 head 完成自己的 Q/K/V 匹配和 Value 聚合，各 head 输出 concat 后再经过 WO。WO 是普通 Linear，它把 concat 的 feature 混合并投回模型宽度 d，所以 Attention 分支可以产生与 Hℓ 相加的 ΔA。

这一节不重做 Q/K/V。这里唯一需要记住的是：Attention 的输入是整个序列，输出却为每个 token 返回同宽度的一行更新。

MLP 分支：每行复用同一个非线性函数

F-2.5 的 MLP trace 已把一条向量走完：Linear 1 把 d 扩到 dff，GELU 逐元素加入非线性，Linear 2 再回到 d。这里把它看作产生 ΔM 的函数：

$$MLP(x)=GELU(xW_1+b_1)W_2+b_2$$

同层所有 token 使用同一组 W1、b1、W2、b2。共享的是函数，不是输出：不同位置输入不同，得到的 ΔM 也可不同。GELU 的正负响应、bias 和训练细节请回到 F-2.6。

Attention 负责通信、MLP 负责逐位置计算是一个有用但不排他的分工。不同层的 MLP 参数不同；不能把某一层的某个维度机械命名为固定语义。

Residual：原表示加上分支更新

抽象写作 $y=x+F(x)$。x 是进入某个子层前的 hidden state，F(x) 是 Attention 或 MLP 分支算出的更新，y 是相加后的结果。把 F(x) 叫 Δ 只是方便：它表示“在当前表示上加什么”，不保证数值一定小。

两条前向路径在加号处汇合
x ───────────────┐
                 + → y = x + F(x)
x → 子层 F(x) ──┘

直观上，网络不必每层从头写出一个新表示；它可以先保留已有状态，再学习如何补充或修改。没有 residual 也可以定义网络，但深层堆叠更容易把早先信息完全交给后续变换处理。Residual 提供了一条明确的 identity 项，却不能保证信息永不丢失，也不保证整个 Block 可逆。

optional：为什么又有人说“梯度高速公路”

前向的“原表示旁路”与反向的“梯度旁路”相关，但不是同一句话。对 $y=x+F(x)$ 求导会出现 identity 项：

$$\frac{\partial y}{\partial x}=I+J_F(x)$$

这里 I 是恒等映射，$J_F$ 是 F 的 Jacobian。直觉上，反向信号除了穿过复杂分支外，还看到一条直接的恒等贡献。它通常有利于深网络优化，但不能保证梯度永不消失、爆炸或训练成功；这段是 optional 微积分补充，不是理解 Block 主线的前提。

LayerNorm：让进入分支的数值更稳定

多层相加和变换会让各 token 向量的数值尺度持续变化。LayerNorm 在送入 Attention 或 MLP 分支前，先把当前 token 自己的 d 个 feature整理到较稳定的尺度，再用可学习参数恢复模型需要的缩放和平移。

$$LN(x)=\gamma\odot\frac{x-\mu}{\sqrt{\sigma^2+\epsilon}}+\beta$$

在这条式子里，μ 是该 token 的 feature 均值，σ2 是同一行的 feature 方差，ε 是避免除零的小常数；γ、β 是每个 feature 可学习的缩放与平移。它们都在本 LayerNorm 内部定义，不会跨 token 偷看数据。

它只沿最后一个 feature 轴归一化

对张量 X[B,n,d]，LayerNorm 分别处理每个 X[b,i,:]。它不跨 batch，也不跨 token；“她”的均值和方差来自“她”自己的 d 个 feature，“苹果”另算一套。

$$\mu_{b,i}=\frac{1}{d}\sum_{k=1}^{d}X[b,i,k]$$
$$\sigma^2_{b,i}=\frac{1}{d}\sum_{k=1}^{d}(X[b,i,k]-\mu_{b,i})^2$$

这也是为什么 LayerNorm 可以在 batch size 为 1 时照常工作。它和 BatchNorm 按 batch 聚合统计量的思路不同。

LayerNorm 改数值，为什么不等于抹掉语义？

它确实会改变一行向量的均值和尺度，因此不能说“原向量被原样存回去”。但它保留了同一行 feature 相对偏离均值的结构，并允许 γ（gamma）、β（beta）学习每个 feature 合适的缩放和平移。后续 Attention、MLP 和 residual 也继续共同决定表达。

LayerNorm 通常是非可逆的；不能从归一化结果唯一恢复原始均值和方差。它的目的不是无损压缩，而是让分支接收更稳定的数值分布。

同一个想法有常见变体

变体最小区别本节怎么读
原始 post-norm先做子层和 residual，再 LayerNorm。与本节主线顺序不同。
现代 pre-norm先 LayerNorm，再做子层并加回。本节公式采用这一种。
RMSNorm以 RMS 缩放，常不减均值。仍是稳定分支输入的归一化设计。
SwiGLU用门控 MLP 代替简单 GELU MLP。仍是逐位置的非线性计算。

为什么重复几十层？是反复 refinement，不是固定语义楼层

每一层的通信和更新成为下一层输入，再由一套新参数继续处理。层数增加，模型有机会把局部关系、长程关系和任务相关特征反复组合；但不能强行规定“第 3 层一定是语法、第 12 层一定是语义”。不同模型、数据和任务会改变这种趋势。

最后的 hidden state 怎样用于不同任务

整个模型训练的核心，是让每层更新后的 hidden states 越来越适合最终任务头读取。Loss 通过任务头和所有 Block 反向传播；冻结 backbone、只训练 adapter/LoRA，或换用不同任务头时，实际更新范围会不同。

系统任务头常读什么输出例子
GPT当前位置 hidden state下一个 token 的 logits。
BERT被 mask 的位置或 pooled 表示词表预测、分类。
VLM融合后的语言/视觉 hidden states文本 token、匹配或分类。
VLA视觉、语言、状态融合后的表示动作 token、连续 action 或 action chunk。

检查：把常见误解改写准确

  1. “Attention 后直接预测就够了，所以 MLP 多余。”不准确：可以直接接头，但 MLP 为每个位置增加了逐位置非线性计算能力。
  2. “Residual 会原样保存所有信息。”不准确：它提供 identity 项和直接路径，不能保证可逆或绝不丢失信息。
  3. “LayerNorm 在全 batch 或全句子上算均值。”不准确：它对每个 X[b,i,:] 独立沿 d 计算。
  4. “跟踪一条 token 就代表 Attention 只处理这条 token。”不准确：我们只跟踪焦点输出；Attention 仍读取整个可见序列。
  5. “层越深就有固定、可命名的语义职责。”不准确：深层是反复 refinement，趋势不等于硬规则。

带走这一条连续主线

Attention 负责通信，MLP 负责逐位置计算，Residual 保留并传递当前表示再叠加更新，LayerNorm 稳定进入分支的数值。一个 Block 用两次“规范化 → 分支 → 加回”把 [B,n,d] 更新成新的同 shape hidden states；堆叠多个 Block 后，任务头再把它读成 GPT、BERT、VLM 或 VLA 所需的输出。
