Transformer Block 里到底有什么参数

你经常听到“3B 模型”“7B 参数”。这些参数到底花在哪里？一个 Transformer Block 主要就干两件事：让信息在 token 之间流动（Attention），再让每个 token 自己加工（MLP）。理解这个拆分，对后面看 Action Expert 为什么独立一套权重非常关键。

上一节你理解了 Attention 在做什么。这一节回答：这些计算在模型里占哪块、参数花在哪。提醒：Block 处理的是 token 向量序列（每个 token 是什么、f-4 会讲）；这里先假设已经有一排向量了。

两层楼梯的比喻

把一个 Block 想象成两层楼梯：

  1. 第一层（Attention）：大家坐在一起开会，互相交换信息。
  2. 第二层（MLP）：每个人回到座位上，自己消化刚才听到的内容。

真实 Block 还多了 残差连接（$x + \text{子层}(x)$，防信息丢）和 LayerNorm（稳定训练）。可以记成：开会（Attention）→ 加残差、归一化 → 独自消化（MLP）→ 再加残差、归一化。下面先拆两块主菜。

Attention 部分（信息交换）

主要参数是四个线性投影：Q、K、V、以及输出投影。它们负责“谁该看谁”。

四个矩阵中，前三个把每个 token 的向量投影成 Q、K、V（对应 f-2）；第四个把 Attention 输出映射回模型维度。参数量通常小于同层的 MLP，但 决定了谁看谁。

MLP 部分（信息加工）

通常是两个大矩阵（先升维、再降维），参数量往往占整个 Block 的 2/3 以上。它的任务是对每个 token 做非线性变换。

直觉：Attention 负责 横向交流；MLP 负责 每个 token 自己变聪明一点。没有 MLP 的非线性，多层 Attention 叠起来仍偏线性，表达能力不够。

从 RL 看 为什么 MoE 爱改 MLP

Attention 像“大家共享的会议室”，MLP 像“每个人的私人笔记本”。给不同专家单独的 MLP，就等于给不同“人”不同的笔记本，而会议室还是同一个。这就是 Action Expert 独立 MLP 的直觉来源。

这一节你需要带走的

一个 Block = Attention（跨 token 通信） + MLP（单 token 加工）。MLP 参数更多，也更容易被替换成专家（MoE）。后面 π0 的 action expert 正是独立了一套 MLP。下一节补上一块拼图：图像、语言、动作怎么变成 token 向量——否则你不知道这排向量从哪来。
