为什么会有 Transformer？

在开始讲 VLA 之前，我们先回答一个更根本的问题：为什么今天几乎所有大模型（包括 VLA）最后都长成了 Transformer 的样子？ 理解了这一点，后面的 Attention、Token、Action Expert 才不会像魔法。

序列处理的古老难题

很长一段时间，人们处理序列（文字、语音、轨迹）用的是 RNN（循环神经网络）。它的核心想法是：一个一个 token 读过去，每读一个就把信息记在“隐藏状态”里。

这个想法听起来很自然，但有两个致命问题：

  • 顺序计算极慢：第 1000 个词必须等前 999 个词全部算完。
  • 长距离遗忘：越早的信息在反复传递过程中会逐渐衰减。处理一段 50 步的机器人动作时，t=1 的状态信息到 t=50 时几乎没了。

Self-Attention 的革命

2017 年，Attention Is All You Need 提出： 不要一个一个读，而是一次性让所有 token 互相看。

这样做的两个巨大好处：

  • 计算可以完全并行（GPU 喜欢并行）。
  • 任意两个 token 之间的距离永远是 1 步，不存在“远距离遗忘”。
一句话记住 Transformer 的本质

Transformer = 一堆 token + 让它们互相通信的机制 + 让每个 token 自己加工信息的机制。

为什么 VLA 也必须用它

机器人动作是一段连续的序列（action chunk）。图像是空间序列（patches）。语言是指令序列。把这三者放在一起，最自然的模型就是能同时处理多模态序列、并让它们互相通信的架构——Transformer。

这一节你需要带走的

(1) RNN 顺序慢 + 遗忘远距离；(2) Self-Attention 让所有 token 一次性互相看，彻底解决这两个问题；(3) VLA 里的图像、语言、状态、动作最终都要变成 token 序列，因此 Transformer 是目前最自然的骨架。
