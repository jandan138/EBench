Attention 从零开始（Q/K/V 直觉）

如果你看到 $QK^T$ 就懵，通常不是你不适合学，而是中间少了几级台阶。这一节把 Attention 拆成小白可跟的五步：token 是向量，点积表示相似，Q/K/V 是同一个 token 的三种角色，softmax 把分数变成权重，最后按权重混合 Value。

先看一句人话：“她在指小红”

假设输入是：

“她 在 指 小红 的 方向”

当模型读到“她”时，它不能只看“她”这个字。它要问：这个“她”指谁？比较合理的线索是后面的“指”和“小红”。人类会自动把“小红”的信息补到“她”上；Attention 就是在模型内部做这个补信息的过程。

上一节的圆桌会议比喻到这里变具体：每个 token 都可以发问，也可以被别人查询。Attention 不会给一个硬答案，而是给一排软权重：看“小红”多一点，看“方向”少一点，看“在”更少一点。

interactiveAttention 交互演示
点击不同词切换 Query。先点“她”，再点“小红”，观察每个词看别人的权重如何变化。数值是教学示意，不是训练出的真实权重。

第一层台阶：token 先变成向量

模型不是直接处理汉字。每个 token 进入 Transformer 前，都会变成一个向量。为了能手算，先假装每个词只有 2 维：

token向量示意直觉
她[0.8, 0.1]更像“人物指代”
指[0.4, 0.7]更像“动作关系”
小红[0.9, 0.2]更像“具体人物”

向量之间可以做点积。点积越大，粗略表示越相似、越匹配。比如 [0.8, 0.1] 和 [0.9, 0.2] 的点积是 $0.8 \times 0.9 + 0.1 \times 0.2 = 0.74$，比和很多无关词的点积更高。

第二层台阶：为什么要 Q/K/V 三套向量

现在问题来了：同一个 token 只有一条向量不行吗？不太够。因为一个 token 在 Attention 里同时有三种职责：

名字中文直觉它在问什么例句里的角色
Query问题我现在想找什么信息？“她”发出问题：我指谁？
Key标签/索引我能被什么问题匹配到？“小红”的 Key 像名牌：我能回答人物指代。
Value内容如果别人关注我，我贡献什么信息？“小红”的 Value 是具体人物语义。

Q/K/V 不是人工给词贴的标签，也不是三份不同输入。它们是同一个输入向量经过三套可学习矩阵变出来的三种角色。

$$q_i = x_i W_Q,\qquad k_i = x_i W_K,\qquad v_i = x_i W_V$$

这里 $x_i$ 是第 $i$ 个 token 的原始表示，$W_Q,W_K,W_V$ 是模型训练出来的参数。训练足够多之后，模型会学会让“人物指代”的 Query 更容易匹配到“具体人物”的 Key。

第三层台阶：先看一个 Query 怎么看所有 Key

先不要看矩阵。只看“她”这一个 Query。它会和所有 token 的 Key 算匹配分数：

$$s_{\text{她},j} = \frac{q_{\text{她}} \cdot k_j}{\sqrt{d_k}}$$

$j$ 会扫过“她、在、指、小红、的、方向”。得到的不是最终答案，而是一排分数。比如教学示意里，“她”对三个 token 的分数可能是：

Key她指小红
score1.00.02.0
softmax 后权重0.2450.0900.665

这说明“她”主要看“小红”，也保留一点自信息。权重加起来等于 1。最后，“她”的新表示就是按这些权重混合各 token 的 Value：

$$z_{\text{她}} = \sum_j \alpha_{\text{她},j} v_j$$

第四层台阶：矩阵公式只是把所有 Query 一次算完

如果序列里有 $n$ 个 token，每个 token 都要当一次 Query。逐个写太慢，所以用矩阵一次性写完：

$$Q,K,V \in \mathbb{R}^{n \times d_k},\qquad QK^T \in \mathbb{R}^{n \times n}$$

$QK^T$ 的第 $i$ 行表示“第 $i$ 个 token 看所有 token 的原始分数”。第 $j$ 列表示“所有 token 看第 $j$ 个 token 的分数”。softmax 是按行做的，因为每一行都要变成“某个 Query 看全体 Key 的权重分布”。

$$\text{Attention}(Q,K,V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

把这行公式翻译成人话：

  • $QK^T$：所有人拿自己的问题，去匹配所有人的标签。
  • $\sqrt{d_k}$：缩放分数，避免维度大时点积过大。
  • softmax：每一行分数变成权重，合计为 1。
  • 乘 $V$：按权重把内容混合回来，得到每个 token 的新表示。

第五层台阶：mask 决定“能看谁”

有时不是所有 token 都允许互相看。语言模型从左到右生成时，当前位置不能偷看未来 token。这时会在 softmax 前把禁止看的分数改成极小值，softmax 后权重近似 0。

mask 类型能看谁后文在哪里出现
Causal mask只能看当前和过去。自回归生成下一个词、下一个动作 token。
Bidirectional mask一段已知 token 内部可以互相看。图像/指令编码，或 action chunk 内部的双向通信。
Block mask某些模态之间允许看，某些不允许。Part 5 的 Transfusion / action expert 会用到。
从 RL 看 Attention 像软检索记忆

策略输出动作前，可能要参考当前图像、指令、历史动作、状态。Query 像当前决策点发出的检索请求，Key 像每段信息的索引，Value 是真正取回的内容。不同的是，这个检索不是手写规则，而是通过训练学出来的。

小检查：看到 QKV 时怎么不慌

请你在脑子里把公式拆成三句话：

  1. 每个 token 先从 $x_i$ 变出 $q_i,k_i,v_i$。
  2. 某个 $q_i$ 和所有 $k_j$ 点积，得到“我该看谁”的分数。
  3. softmax 后按权重混合所有 $v_j$，得到这个 token 的新表示。
常见误解

Q/K/V 不是“问题词、关键词、数值词”三类 token。每个 token 都同时有自己的 Q、K、V。比如“小红”既可以作为别人查询的 Key/Value，也可以作为 Query 去看“她”和“方向”。

这一节你需要带走的

Attention 的本质不是公式，而是“按相关性取信息”。你现在应该能把 $QK^T$ 看成一张 $n \times n$ 的关系表：每一行是一个 Query 看所有 Key 的分数。下一节 F-3 会把 Attention 放回完整 Transformer Block，看看除了 Q/K/V，模型里还有哪些参数。
