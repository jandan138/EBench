Attention 从零开始（Q/K/V 直觉）

如果你看到 $QK^T$ 就懵，通常不是你不适合学，而是中间少了几级台阶。这一节把 Attention 拆成一条小白可跟的链路：token 是向量，点积表示相似，Q/K/V 是同一个位置的三种投影角色，softmax 把分数变成权重，最后按权重混合 Value。

先看一句人话：“她在指小红”

假设输入是：

“她 在 指 小红 的 方向”

当模型读到“她”时，它不能只看“她”这个字。它要问：这个“她”指谁？比较合理的线索是后面的“指”和“小红”。人类会自动把“小红”的信息补到“她”上；Attention 就是在模型内部做这个补信息的过程。

上一节的圆桌会议比喻到这里变具体：每个 token 都可以发问，也可以被别人查询。Attention 不会给一个硬答案，而是给一排软权重：看“小红”多一点，看“方向”少一点，看“在”更少一点。

先说明：Query 不是一个真的中文问题

“它要问”只是比喻。数学上，Query 是当前 hidden state 乘 $W_Q$ 得到的一条向量；Key 是 hidden state 乘 $W_K$ 得到的一条向量；二者点积给出匹配分数。没有任何 token 真的在发问，GPU 只是在做矩阵乘法。

interactiveAttention 交互演示
点击不同词切换 Query。直觉模式看权重变化；手算模式会展示 toy Q/K/V、点积 score、scaled score、softmax 和 weighted Value。数值是教学示意，不是训练出的真实权重。

第一层台阶：Attention 的输入是当前 hidden state

模型不是直接处理汉字。每个 token 进入 Transformer 前，会先变成初始向量；进入第 $\ell$ 层 Attention 时，真正输入的是这一层的 hidden states，常写成 $X^\ell$。

$$X^\ell = [h_1^\ell,\ldots,h_n^\ell] \in \mathbb{R}^{n \times d}$$

为了能手算，先假装某一层里每个词的 hidden state 只有 2 维：

token向量示意直觉
她[0.8, 0.1]更像“人物指代”
指[0.4, 0.7]更像“动作关系”
小红[0.9, 0.2]更像“具体人物”

这里的向量如果在第 0 层，接近 embedding 加位置信息；如果在更深层，已经混入了前面层的上下文。Attention 每层都在更新这些 hidden states。

为什么点积能当匹配分数

两个向量 $a,b$ 的点积是：

$$a\cdot b = \|a\|\,\|b\|\cos\theta$$

如果向量长度差不多，点积主要看夹角 $\theta$。方向越接近，$\cos\theta$ 越大，点积越大；方向接近垂直，点积接近 0；方向相反，点积可能为负。所以点积可以当作一种“方向是否对得上”的匹配分数。

但要小心：它不是天然语义相似度。真正学到“谁该匹配谁”的，是 $W_Q,W_K$ 把 hidden state 投到什么几何空间里。训练前，这个空间基本随机；训练后，模型才逐渐学出有用的方向。

第二层台阶：为什么要 Q/K/V 三套向量

现在问题来了：既然已经有 hidden state，为什么还要再变出 Q/K/V 三套向量？因为同一条 hidden state 在 Attention 里要承担三种不同的计算用途：拿来匹配别人、被别人匹配、以及被加权汇聚。如果只用一套向量，三件事会被绑死。

名字中文直觉它在问什么例句里的角色
Query问题我现在想找什么信息？“她”发出问题：我指谁？
Key标签/索引我能被什么问题匹配到？“小红”的 Key 像名牌：我能回答人物指代。
Value内容如果别人关注我，我贡献什么信息？“小红”的 Value 是具体人物语义。

Q/K/V 不是人工给词贴的标签，也不是三份不同输入。它们是同一条 hidden state 经过三套可学习矩阵投影出来的三组工作向量：

$$q_i = h_i^\ell W_Q,\qquad k_i = h_i^\ell W_K,\qquad v_i = h_i^\ell W_V$$

$W_Q,W_K,W_V$ 是普通参数矩阵，训练开始时通常随机初始化。刚初始化时，它们不会“找主语”“找宾语”。只有在大量训练样本和 loss 反向传播推动下，它们才会慢慢变成有用的投影：让该互相匹配的 query/key 在投影空间里点积更大，让不该匹配的点积更小。

Q/K/V 有没有全局信息？

取决于 $h_i^\ell$ 本身。第一层的 $h_i^0$ 主要来自 embedding 和位置，几乎还没有跨 token 上下文；更深层的 $h_i^\ell$ 已经经过前面层的 Attention，可能带有上下文。无论哪一层，真正把“这一层的所有可见 token 信息”汇入第 $i$ 个位置，是后面的加权 Value：$z_i=\sum_j \alpha_{ij}v_j$。

第三层台阶：先看一个 Query 怎么看所有 Key

先不要看矩阵。只看“她”这一个 Query。它会和所有 token 的 Key 算匹配分数：

$$s_{\text{她},j} = \frac{q_{\text{她}} \cdot k_j}{\sqrt{d_k}}$$

$j$ 会扫过“她、在、指、小红、的、方向”。得到的不是最终答案，而是一排分数。教程或 widget 里的“她 4、指 1、小红 7”这类数字，本质上就是 $q_{\text{她}}$ 分别和每个 $k_j$ 点积后得到的 score。下面给一个完整玩具计算。

对象向量点积
$q_{\text{她}}$[1, 2]-
$k_{\text{她}}$[2, 1]$1\times2+2\times1=4$
$k_{\text{指}}$[1, 0]$1\times1+2\times0=1$
$k_{\text{小红}}$[3, 2]$1\times3+2\times2=7$

这些分数可以很大也可以为负，不能直接当“比例”。先除以 $\sqrt{d_k}$ 做缩放。这里 $d_k=2$，所以 $\sqrt{d_k}\approx1.414$：

Key她指小红
raw score417
scaled score2.830.714.95
softmax 后权重约 0.106约 0.013约 0.881

如果不除以 $\sqrt{d_k}$，raw score 进入 softmax 会更尖，最大项更容易接近 1，其他项接近 0。维度越高，点积方差越大，softmax 越容易饱和，训练梯度也更不稳定。缩放不是语义技巧，而是数值稳定技巧。

注意：这一节只算了 Query 和 Key 的匹配分数。Value 还没出场——它会在 softmax 得到权重之后，作为“被取走的内容”参与最后一步混合。先别急着问“为什么不直接混合 Key”，等算完权重再看下一节。

Softmax：把分数变成可加权的比例

Softmax 的输入是一行 scores，输出是一行非负权重：

$$\alpha_{ij}=\frac{\exp(s_{ij})}{\sum_m \exp(s_{im})}$$

为什么不用“每个 score 直接除以总和”？因为 score 可能是负数，总和也可能接近 0，直接归一化会得到负权重或不稳定比例。指数函数有三个好处：输出永远为正；高分会被放大得更多；整个函数可微，方便反向传播。Softmax 叫“soft max”，是因为它像一个可微的近似最大选择：最高分权重大，但不是硬选唯一一个。

softmax 后权重一定加起来等于 1，因为分母就是所有指数值的总和。因此这一行权重可以理解成“第 i 个位置从各个位置取信息的比例”。这不是概率的唯一解释，但作为 attention weight 很自然。

到这里为止，我们只完成了“找谁”：Query 和 Key 点积 → 缩放 → softmax，得到一排权重 α。还记得第二层台阶里的 Value 吗？它在公式 v_i = h_i W_V 里已经出现过，但手算例子里还没用上。权重只是比例，真正要被混合进去的内容，在 Value 里。 下一节回答：为什么混合 Value，而不是 Key。

第四层台阶：用权重混合 Value（为什么不混合 Key）

把 Attention 想成一条四步流水线，到这里你已经走完了前三步：

  1. Query：当前位置想问什么（例：“她”指谁？）
  2. Key：每个位置能被怎么匹配（例：“小红”的 Key 像名牌）
  3. softmax：把匹配分数变成权重 α（例：小红 ≈ 0.881）
  4. Value：按权重把各位置的内容取回来（这一步现在才手算）

Key 的职责是“被匹配”，Value 的职责是“被取走”。这两件事分开很重要。一个位置可以用某种索引特征让别人找到自己，但真正贡献给别人的内容可以是另一组特征。数据库里也类似：索引用来查找，记录内容才是读出来的数据。

假设三个 Value 是（和上面的 Q/K 一样，也是 hidden state 乘 W_V 得到的；这里直接给出结果方便手算）：

Value向量
$v_{\text{她}}$[1, 0]
$v_{\text{指}}$[0, 1]
$v_{\text{小红}}$[3, 1]

那么“她”的新上下文向量就是：

$$z_{\text{她}} = \sum_j \alpha_{\text{她},j} v_j$$

代入上面的权重，得到大约：

$$z_{\text{她}}\approx0.106[1,0]+0.013[0,1]+0.881[3,1]=[2.749,0.894]$$

这一步之后，第“她”这个位置的 hidden state 已经混入了“小红”的 Value，所以它开始拥有当前层可见范围内的上下文。后面还会经过输出投影、残差、LayerNorm、MLP，形成下一层 hidden state——这些在下一节 F-3 会完整展开（见 输出投影、残差与 LayerNorm、MLP 前置章）。

矩阵公式：把所有 Query 一次算完

如果序列里有 $n$ 个 token，每个 token 都要当一次 Query。逐个写太慢，所以用矩阵一次性写完：

$$Q,K,V \in \mathbb{R}^{n \times d_k},\qquad QK^T \in \mathbb{R}^{n \times n}$$

$QK^T$ 的第 $i$ 行表示“第 $i$ 个 token 看所有 token 的原始分数”。第 $j$ 列表示“所有 token 看第 $j$ 个 token 的分数”。softmax 是按行做的，因为每一行都要变成“某个 Query 看全体 Key 的权重分布”。

$$\text{Attention}(Q,K,V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

把这行公式翻译成人话：

  • $QK^T$：所有 query 同时和所有 key 点积，得到 $n\times n$ 的 score 矩阵。
  • $\sqrt{d_k}$：缩放分数，避免维度大时点积过大。
  • softmax：每一行分数变成权重，合计为 1。
  • 乘 $V$：按权重把内容混合回来，得到每个 token 的新表示。

mask：决定“能看谁”

有时不是所有 token 都允许互相看。语言模型从左到右生成时，当前位置不能偷看未来 token。这时会在 softmax 前把禁止看的分数改成极小值，softmax 后权重近似 0。

mask 类型能看谁后文在哪里出现
Causal mask只能看当前和过去。自回归生成下一个词、下一个动作 token。
Bidirectional mask一段已知 token 内部可以互相看。图像/指令编码，或 action chunk 内部的双向通信。
Block mask某些模态之间允许看，某些不允许。Part 5 的 Transfusion / action expert 会用到。
从 RL 看 Attention 像软检索记忆

策略输出动作前，可能要参考当前图像、指令、历史动作、状态。Query 像当前决策点发出的检索请求，Key 像每段信息的索引，Value 是真正取回的内容。不同的是，这个检索不是手写规则，而是通过训练学出来的。

小检查：看到 QKV 时怎么不慌

请你在脑子里把公式拆成三句话：

  1. 每个位置先从当前 hidden state $h_i^\ell$ 变出 $q_i,k_i,v_i$。
  2. 某个 $q_i$ 和所有 $k_j$ 点积，得到“我该看谁”的分数。
  3. softmax 后按权重混合所有 $v_j$，得到这个 token 的新表示。
常见误解

Q/K/V 不是“问题词、关键词、数值词”三类 token。每个 token 都同时有自己的 Q、K、V。比如“小红”既可以作为别人查询的 Key/Value，也可以作为 Query 去看“她”和“方向”。

这一节你需要带走的

Attention 的本质不是公式，而是“按相关性取信息”。你现在应该能把 $QK^T$ 看成一张 $n \times n$ 的关系表：每一行是一个 Query 看所有 Key 的分数。下一节先到 F-2.5 补齐 Linear、GELU 与 MLP，再进入 F-3 的完整 Transformer Block。
