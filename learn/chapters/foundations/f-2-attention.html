Attention 从零开始（Q/K/V 直觉）

Transformer 里最重要、也最容易被“数学符号”吓跑的一块，就是 Self-Attention。这一节我们完全不讲公式推导，先用一个最日常的语言例子，让你产生强烈的第一感觉：模型是怎么“知道”哪个词该关注哪个词的。

上一节我们说 Transformer 的核心是 让所有 token 一次性互相看。这一节就把「互相看」拆成三步：谁想问（Query）→ 和谁匹配（Key）→ 拿走什么信息（Value）。先读例子，再玩下面的交互；不用背公式。

“她在指小红”的直觉

想象一句话：

“她 在 指 小红 的 方向”

当模型读到“她”的时候，它需要回答一个问题：“她到底是谁？”

人类大脑会下意识地往后看，找到“指”和“小红”，然后把“小红”这个信息和“她”联系起来。

Attention 做的就是同一件事——让每个词去“提问”其他词，看看谁最能回答它的问题。

读「她」时，大脑其实在问：「她」指谁？ 然后扫向「指」「小红」。Attention 把这件事形式化：每个词都带一个「问题」，去和其他词的「标签」比一比，看谁最像答案。

你刚才点的词就是 Query（提问者）；柱形越高，说明这个词认为对方越能回答自己的问题。这些权重不是人手工标的，是模型用三套向量算出来的。下面给每个词配上这三套「身份」。

Query、Key、Value：三个物理角色

为了实现上面的“提问-匹配-拿信息”，模型给每个词准备了三套向量：

  • Query（问题）：这个词“我想问什么？”
  • Key（标签）：其他词“我能回答什么问题？”
  • Value（内容）：其他词“我真正包含的信息”

回到例句：读「她」时，「她」发 Query（「我是谁？」）；「小红」的 Key 像名牌（「我能回答『指谁』」）；「小红」的 Value 是实际语义（「小红这个人」）。Attention 用 Query 和所有人的 Key 算权重，再按权重混合所有人的 Value——「她」的新表示就混进了「小红」的信息。

从 RL 看 像一个带记忆的策略

（不熟悉强化学习可跳过本框，不影响后文。）

Query 像当前状态下 policy 发出的“查询请求”；Key 像不同记忆片段的索引；Value 是真正要取出来的动作/信息。Attention 就是在做一个可学习的、软的“记忆检索”。

最核心的一行数学（只看直觉）

每个词的向量先经过三个可学习的矩阵，变成 Q、K、V（这就是 f-3 里说的「四个线性投影」里的前三个）。公式只是把「提问—匹配—加权取内容」写成一行。

$$\text{Attention}(Q,K,V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d}}\right)V$$

逐项翻译：

  • $QK^T$：每个词的问题和所有词的标签做匹配，得到“相关性分数”。
  • 除以 $\sqrt{d}$：防止分数太大导致 softmax 梯度消失（工程细节）。
  • softmax：把分数变成“概率权重”（所有词的关注度加起来是 1）。
  • 乘 V：按权重把真正的内容混合起来。

上式默认每个词能看见序列里所有词。但 生成下一个词时不能偷看未来——所以在 $QK^T$ 上盖一层 mask（把不允许看的位置设为 $-\infty$），softmax 后权重为 0。下面分两种常见 mask。

causal vs bidirectional：两种“能看谁”

Causal（因果）：语言模型从左到右生成，位置 $i$ 只能看 $\le i$ 的词 → causal mask（下三角）。

Bidirectional（双向）：编码器或已整段给定的 action chunk 里，token 可以互相看。VLA 里常见组合：看图/读指令用双向，自回归出动作 token 用因果；action chunk 内部的动作 token 经常是 full bidirectional，因为「一整段动作」可以互相参考。

这一节你真正需要记住的画面

每个 token 都同时扮演“提问者”和“被问者”。Query 负责问问题，Key 负责被匹配，Value 负责贡献信息。Attention 就是让模型在序列里做一次可学习的、软的“谁该看谁”。

这一节你需要带走的

(1) Attention 的本质是“让每个词去问其他词谁最能回答它”；(2) Q/K/V 是三个不同角色的向量；(3) 权重 = softmax(QK^T)；(4) causal mask 保护生成顺序，bidirectional 让 chunk 内部互相看。下一节我们看这些计算发生在 Transformer Block 里的哪一层。
