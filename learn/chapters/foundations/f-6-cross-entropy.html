Cross Entropy 与 Softmax

F-5 的 MSE 是“预测一个连续向量”。Cross Entropy 是另一种常见训练目标：在有限选项里，把概率质量推到正确选项上。LLM 训练下一个词靠它；当动作被 FAST 这类 tokenizer 离散成 action token 时，动作训练也能借用这套机制。

第一步：模型先吐 logits，不是概率

分类模型最后通常输出一排原始分数，叫 logits。比如模型要从 5 个动作词里选一个：

候选 tokenpickplacepushwaitother
logit2.11.40.80.3-0.5

logit 可以是任意实数，不要求大于 0，也不要求加起来等于 1。它只是“模型偏好”的原始分数。要把它变成概率，就用 softmax。

第二步：Softmax 把分数变成概率

$$p_i = \frac{\exp(z_i)}{\sum_j \exp(z_j)}$$

$z_i$ 是第 $i$ 个 logit。指数函数会放大高分和低分的差距；分母把所有选项归一化，让概率加起来等于 1。实际代码里会先减去最大 logit，避免指数爆炸，但数学含义不变。

interactiveSoftmax 温度：概率分布如何变尖或变平
温度是推理/采样时常见的旋钮。训练 Cross Entropy 时通常按标准 logits 计算，不靠调高温度来“修”损失。

第三步：Cross Entropy 只盯正确类概率

如果正确答案是类别 $c$，Cross Entropy 就是：

$$\mathcal{L}_{\text{CE}} = -\log p_c$$

正确类概率越高，损失越小。几个数感受一下：

正确类概率 $p_c$CE = $-\log p_c$直觉
0.900.105模型很确定，罚得小。
0.500.693模型半信半疑。
0.014.605正确答案几乎没概率，罚得很重。

如果用 one-hot 标签 $y_i$ 写，公式是：

$$\mathcal{L}_{\text{CE}} = -\sum_i y_i \log p_i$$

因为 one-hot 里只有正确类位置是 1，所以它等价于 $-\log p_c$。

序列训练：每个位置都做一次分类

LLM 训练不是只选一个 token，而是在每个位置预测下一个 token。常见 shape 是：

$$\text{logits}\in \mathbb{R}^{B \times T \times V},\qquad \text{target}\in \mathbb{R}^{B \times T}$$

$B$ 是 batch size，$T$ 是序列长度，$V$ 是词表大小。每个位置都有一排 $V$ 维 logits，对应一个正确 target id。序列版 CE 就是把所有有效位置的 $-\log p(\text{target})$ 平均起来。

动作 token 为什么也能用 CE

连续动作本来不能直接分类。但如果先把动作量化成有限词表里的 token，任务就变成“下一个 action token 是哪一个”。这就是 FAST 等 action tokenizer 思路背后的训练便利性：把部分动作学习问题改写成和语言建模非常相似的离散 token 预测。

别混淆两个 softmax

F-2 Attention 里的 softmax 是把“某个 Query 看所有 Key 的分数”变成注意力权重。这里的 softmax 是把“词表 logits”变成下一个 token 的概率。数学操作相同，语义不同。

小检查：手算方向感

如果三个 logits 是 $[2,1,0]$，softmax 后第一个类别概率最大。如果正确答案是第一个类别，CE 比较小；如果正确答案是第三个类别，CE 会大很多。你不需要每次精确心算，但要有这个方向感：CE 惩罚的是“正确 token 被模型放了多少概率”。

从 RL 看 离散 action 和连续 action 的分叉

如果动作空间是几个按钮，策略可以输出 categorical distribution，用 CE 或 policy gradient 都很自然。如果动作是机械臂连续控制，直接分类不够，需要回归、分布建模或先量化成 action token。
