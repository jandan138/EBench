Linear 与 MLP 从零开始

Attention 让一个位置从其他位置取得上下文。接下来只追一件事：这条位置自己的 hidden state 怎样经过同一个小网络，变成一条新的更新。本章会先走完 Linear 1 → GELU → Linear 2；“正数为什么容易通过、负数到底去哪了”留给下一章专门讲。

先把任务说清楚

一条 hidden state 是一排数字。MLP 是一个逐位置网络：每个 token 的当前向量都经过同一套参数，依次执行 Linear 1 → GELU → Linear 2，得到另一排数字。它不会在 token 位置之间通信；那是 Attention 的工作。

本章的主线

先理解“同一个函数怎样处理不同向量”，再让一条具体 hidden state 从输入走到输出。GELU 在本章只作为中间的固定非线性；它的正负响应、bias、ReLU 对照和训练细节在 F-2.6。

函数：不同输入，同一条规则

函数把输入交给同一条规则。例如 $f(x)=2x+1$：$f(0)=1$，$f(2)=5$，$f(-1)=-1$。输入可以不同，规则没有变，所以输出也会随输入而变。

“所有 token 共用一个 MLP”就是这个意思：它们共享函数的参数，不是共享同一个输入或同一个输出。

一个数字的 Linear

先代入具体数字：取 $x=3$、$w=2$、$b=1$，按“先乘再加”计算 $2(3)+1=7$，所以输出 $y=7$。

把同一规则写成字母就是 $y=wx+b$。$x$ 是输入，$w$ 是权重，$b$ 是偏置。严格数学上有偏置时它是 affine transform，但深度学习库仍常把这个模块叫 Linear。训练会调整 w、b，不是人工给它们指定语法角色。

一条向量的 Linear

若输入是 $x=[x_1,x_2]$，每一个输出 feature 都会读取两个输入 feature，再加自己的 bias。W 是把这些权重排成的矩阵：

$$y_1=x_1w_{11}+x_2w_{21}+b_1$$
$$y_2=x_1w_{12}+x_2w_{22}+b_2$$
$$y_3=x_1w_{13}+x_2w_{23}+b_3$$
$$y_4=x_1w_{14}+x_2w_{24}+b_4$$

这一步只混合同一个位置内部的 feature，不让不同 token 位置通信。看完逐项乘加后，才用矩阵写法简写为 $y=xW+b$。

量shape轴的含义
输入 x[d_in]一项对应一个输入 feature。
权重 WW:[d_in,d_out]行对应输入 feature，列对应输出 feature。
偏置 bb:[d_out]每个输出 feature 有一个 bias。

Linear 做什么，不做什么

Linear 会对 feature 加权组合，却没有自己在 token 位置之间传消息的步骤。它也不是把数字限制到某个固定区间；是否有 bias、输入和输出维度是多少，都是具体层的设计。

先把 GELU 放在正确的位置

Linear 1 算完后会得到一排中间数字。GELU 是夹在两次 Linear 之间的固定、逐元素函数：它对这一排里的每个数字分别做同一条非线性规则，不在 token 位置之间通信，也没有另一套可学习矩阵。

现在只需要把它当作“让中间层不再只是一次普通加权组合”的步骤。为什么它对正负数采取不同规则、为什么 bias 会移动门槛，下一章 F-2.6 再慢慢拆开。

为什么中间需要非线性

若没有 GELU，两次连续 affine transform 可以合并成一次：

$$xW_1+b_1\;\longrightarrow\;(xW_1+b_1)W_2+b_2=xW_*+b_*$$
$$W_*=W_1W_2,\qquad b_*=b_1W_2+b_2$$

因此“Linear 1 再接 Linear 2”通常没有增加新的函数形状。加入 GELU 后，整条映射通常不再是 affine，不能普遍合并成一个 Linear。某些特殊参数组合可能碰巧仍像 affine；重点不是任何一组数字都神奇，而是非线性让网络能对输入空间的不同区域采取不同规则。

把一条 hidden state 走完 MLP

现在终于只跟一条输入走到底。固定输入 $x=[2,-1]$。第一层先扩宽，中间逐元素 GELU，第二层再投回原宽度；这就是 $2 → 4 → 2$。

实际模型里，d 是每条 hidden state 的输入和输出 feature 个数，也叫模型宽度；dff 是 Linear 1 之后的中间 feature 个数，也叫中间宽度。

W1 = [[1, 0, 0.5, 0.25], [-1, 2, 0.5, -0.50]]

B1 = [0, 0, 0, 0]

W2 = [[0.2, 0.4], [0.1, -0.2], [0.3, 0.2], [-0.1, 0.3]]

B2 = [0, 0]

2(1)+(-1)(-1)+0=3 2(0)+(-1)(2)+0=-2 2(0.5)+(-1)(0.5)+0=0.5 2(0.25)+(-1)(-0.50)+0=1
输入[2, -1]
Linear 1[3, -2, 0.5, 1]
GELU[2.996, -0.046, 0.346, 0.841]
Linear 2[0.614, 1.529]

例如最终第一维来自 GELU 后四个数的加权和：$2.996(0.2)+(-0.046)(0.1)+0.346(0.3)+0.841(-0.1)\approx0.614$。这里的负数只是普通数值，后层的负权重也可以让它产生正贡献；不要把“负”读成“负面信息”。

Experiment一条向量怎样穿过 MLP

先在“Linear”和“MLP”视图走完整条链；GELU 的正负门控与训练 trace 在 F-2.6 的独立实验中。

同一 MLP，为什么输出仍可不同

设“她”和“苹果”的当前 hidden state 不同。它们都会套用同一组 W1、B1、W2、B2，就像同一函数 $f$ 同时处理两个不同输入：$f(x_{\text{她}})$ 与 $f(x_{\text{苹果}})$。参数共享不意味着输出相同；只有输入相同、没有随机性时，确定性的同一函数才会给出相同输出。

因此共享 MLP 的准确说法是：每个位置复用同一个计算规则，但由自己的输入决定自己的更新。普通 dense MLP 不在行与行之间交换数据；某些 MoE 架构会选择不同专家，那是另一种设计。

那为什么设计成共享，而不是每个位置各配一套参数？第一是参数效率：序列长度 n 可变，位置专用参数会让模型随 n 膨胀，而共享一套时参数量与 n 无关——这也是为什么它叫 position-wise（逐位置）MLP。第二是泛化：代词、动作、物体需要的「加工」其实是同一类非线性变换，一套参数在不同输入上自然给出不同结果，训练会把它调成对各类输入都合用；这正是一个函数的价值——规则固定，行为随输入变化。分工仍然成立：跨位置该交换的信息已经在 Attention 里交换过了，MLP 只需要把每个位置自己那一份加工好。

Attention 和 MLP 怎样接力

可以先用一条不夸张的分工记忆：Attention 负责让不同位置通信，MLP 负责对每个位置当前拿到的向量做非线性计算。它们不是互斥的“语法模块”和“语义模块”；训练只关心整个系统能否降低 Loss，两个子层会共同塑造 hidden state。

下一章会解释 GELU 为什么像软门控。再下一章的 Transformer Block 会把 Attention 的输出、MLP 的输出以及 residual 和 LayerNorm 放到同一条连续数据流里。

这些参数怎样被训练

W1、B1、W2、B2 都是模型参数。任务头产生预测，预测和目标之间形成 Loss；反向传播计算“每个参数稍微改变会怎样影响 Loss”，优化器再更新参数。训练不会人工指定“第 17 个中间 feature 必须识别宾语”。

同一个 MLP 因为处理不同输入，会在不同样本和 token 上产生不同中间数和不同输出；Loss 汇总这些效果后共同更新那一套共享参数。F-2.6 会用一个很小的 scalar toy 展示一个 gate 的 z 怎样在梯度更新中移动。

检查：能否把 MLP 主线复述出来？

  1. Linear 是同一位置内部的加权组合，不负责 token 间通信。
  2. 两层 Linear 中间没有非线性时，通常可合并成一个 affine transform。
  3. GELU 在这里是逐元素的固定非线性，不是另一套注意力或矩阵。
  4. Linear 1 常把 d 扩到 dff，Linear 2 再回到 d。
  5. 所有 token 共用同一 MLP 参数，但不同输入经过同一函数仍可得到不同输出。
