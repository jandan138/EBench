Linear、GELU 与 MLP 从零开始

Attention 已经让一个位置从其他位置取得上下文。接下来，这一位置自己的向量怎样被普通神经网络继续加工？本节只从加减乘法出发，走完 Linear 1 -> GELU -> Linear 2。

先把任务说清楚

一条 hidden state 是一排数字。MLP 是依次执行 Linear 1 → GELU → Linear 2 的逐位置网络：它只对当前这排数字应用同一套参数和规则，输出另一排数字。

函数：不同输入，同一条规则

函数把输入交给同一条规则。例如 $f(x)=2x+1$：$f(0)=1$，$f(2)=5$，$f(-1)=-1$。输入可以不同，规则没有变，所以输出也会随输入而变。

一个数字的 Linear

先代入具体数字：取 $x=3$、$w=2$、$b=1$，按“先乘再加”计算 $2(3)+1=7$，所以输出 $y=7$。

把同一规则写成字母就是 $y=wx+b$。$x$ 是输入，$w$ 是权重，$b$ 是偏置。严格数学上有偏置时它是 affine transform，但深度学习库仍常把这个模块叫 Linear。训练会调整 $w,b$，不是人工给它们指定语法角色。

一条向量的 Linear

若输入是 $x=[x_1,x_2]$，每一个输出 feature 都会读取两个输入 feature，再加自己的 bias。W 是把这些权重排成的矩阵：

$$y_1=x_1w_{11}+x_2w_{21}+b_1$$
$$y_2=x_1w_{12}+x_2w_{22}+b_2$$
$$y_3=x_1w_{13}+x_2w_{23}+b_3$$
$$y_4=x_1w_{14}+x_2w_{24}+b_4$$

这一步只混合同一个位置内部的 feature，不让不同 token 位置通信。看完逐项乘加后，才用矩阵写法简写为 $y = xW + b$。

量shape轴的含义
输入 x[d_in]一项对应一个输入 feature。
权重 WW:[d_in,d_out]行对应输入 feature，列对应输出 feature。
偏置 bb:[d_out]每个输出 feature 有一个 bias。

Linear 做什么，不做什么

Linear 会对 feature 加权组合，却没有自己在 token 位置之间传消息的步骤。它也不是把数字限制到某个固定区间；是否有 bias、输入和输出维度是多少，都是具体层的设计。

GELU：逐元素的平滑非线性

先看具体数值。输入从负数走到正数时，GELU 让较大的正数接近原样通过，把较大的负数压到接近 0，中间则平滑变化。

精确定义是 $GELU(x)=x\Phi(x)$，其中 $\Phi$ 是标准正态分布的累积分布函数。它对向量中的每个数字逐元素计算：没有可学习矩阵，也不在 token 位置之间通信。

较大的正数接近原样通过，较大的负数被压向 0；GELU 可以得到负输出，输出不被限制在 [0,1]。负半轴的一小段曲线并非单调，这正是平滑曲线的数值边界，不该把它说成阈值开关。

后文 trace 中的 GELU 数字是按这个精确定义做数值求值后显示到三位小数的结果，不是新的符号定义。

常见的 tanh 近似可让实现避免直接计算 $\Phi$，但它近似的是精确定义：

$$\operatorname{GELU}_{\tanh}(x)\approx\frac{x}{2}\left[1+\tanh\!\left(\sqrt{\frac{2}{\pi}}\left(x+0.044715x^3\right)\right)\right]$$

为什么中间需要非线性

两个连续 affine transform 没有 GELU 时可以合并：

$$W_*=W_1W_2$$
$$b_*=b_1W_2+b_2$$

加入 GELU 后，整条映射通常不再是 affine，因此不能为任意参数普遍合并成一个 Linear。某些特殊参数组合仍可能碰巧产生 affine 映射；GELU 也不是唯一可用的非线性。

把一条 hidden state 走完 MLP

固定输入 $x=[2,-1]$。第一层先扩宽，中间逐元素 GELU，第二层再投回原宽度；这就是 $2 → 4 → 2$。

实际模型里，d 是每条 hidden state 的输入和输出 feature 个数，也叫模型宽度；d_ff 是 Linear 1 之后的中间 feature 个数，也叫中间宽度。

B1 = [0, 0, 0, 0]

Experiment一条向量怎样穿过 MLP

同一 MLP，为什么输出仍可不同

标准 dense Transformer 同一层的每个 token 都使用同一组 W_1,b_1,W_2,b_2。不同输入可以得到不同输出；在没有 dropout 等随机性的确定性条件下，相同输入经过同一参数也会得到相同输出。

把多条 token state 叠成批量只是实现策略，不代表它们彼此交换数据。这里的结论只针对 standard dense 层；MoE 会由路由选择不同专家，不一定让所有 token 必经同一个 MLP。

Attention 和 MLP 怎样接力

Attention 的教学分工是跨位置通信：每个位置从其他可见位置取信息。MLP 的教学分工是位置内部计算：每个位置加工当前已有的 feature。这不是绝对能力边界；Attention 后在结构上也可以直接接输出头。

这些参数怎样被训练

从头训练通常从初始化的 W_1,b_1,W_2,b_2 开始；微调常从 checkpoint 开始。反向传播计算损失对参数的梯度，优化器才根据梯度更新当前存在且可训练的参数。冻结参数不会被更新，有些架构会省略 bias。

参数不是人工分配“主语”“动作”等角色。把某些 feature 看成 detector 可以是解释视角，但模型通过数据和优化自行形成分布式表示。

检查：把九件事说准确

  1. 函数用同一条规则处理不同输入。
  2. Linear 是加权和加 bias，矩阵只是简写。
  3. W 的行连输入 feature，列连输出 feature。
  4. GELU 逐元素、可为负，不是 token 通信。
  5. 没有非线性时 affine 可合并；特殊参数是例外。
  6. MLP trace 是 2 → 4 → 2 的两次 Linear 加一次 GELU。
  7. 共享参数不要求不同输入输出相同。
  8. Attention 通信、MLP 计算是有用分工，不是绝对边界。
  9. 反向传播给梯度，优化器更新可训练参数。
