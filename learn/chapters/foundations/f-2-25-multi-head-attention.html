Multi-Head Attention 从零开始（head / concat / WO）

F-2 刚刚把一套 Q/K/V 算到了最后的加权 Value。那其实只是一条 Attention 支路，也就是一个 head。真实 Transformer 通常让多条支路并行看同一批 token，再把结果拼接、混合，最后才交给 residual stream。本节专门补齐 head、concat 和 W_O，再回到 MLP 和完整 Block。

先给一个不会误导人的总画面

把 Multi-Head Attention 看成“同一批当前 hidden states，走过多套不同参数的单头 Attention”。每一套就是一个 head，都完成一次 F-2 的完整链路：变出 Q/K/V，算分数，做 softmax，混合 Value。不同 head 的结果最后才合在一起。

同一个输入 X [B,n,dmodel] 每个 token 当前的 hidden state
→
head 1 W_Q(1), W_K(1), W_V(1) 一套完整单头 Attention
head 2 W_Q(2), W_K(2), W_V(2) 另一套完整单头 Attention
→
concat [B,n,h dhead] 拼同一 token 的 feature
→
WO [B,n,dmodel] 学习如何混合各 head 结果
先把一句话校正

“多个 head 同时看句子”是有帮助的直觉，但不是说 token 真的复制出多个小人。GPU 保存的是张量和参数矩阵；所谓并行，是这些矩阵运算可以沿 head 这一维一起批量执行。

head 在哪条坐标轴上？先分清四个东西

很多困惑来自把不同层级的名词混成一个东西。下面四个词描述的是不同问题：

token / 位置序列里的第几个位置，例如“她”或“小红”。有 n 个。
feature / 通道一个位置的向量里第几个数。有 d_model 个。
Transformer 层第几次完整更新 hidden state。每层有自己的参数。
head同一层 Attention 内的一套独立 Q/K/V 投影和注意力计算。有 h 个。

一个 head 不是 token，不是位置，也不是 Transformer 层。比如“第 3 个 token”“第 3 层”“第 3 个 head”分别在序列轴、深度轴和 Attention 的并行计算轴上；三者不能互换。

先回看：一个 head 本身已经是完整 Attention

令第 r 个 head 的参数带上上标 (r)。它拿到的输入仍然是同一个当前 hidden-state 张量 X，只是用自己的三套矩阵投影：

$$Q^{(r)}=XW_Q^{(r)},\qquad K^{(r)}=XW_K^{(r)},\qquad V^{(r)}=XW_V^{(r)}$$

然后完全照着 F-2 的单头流程走：

$$A^{(r)}=\operatorname{softmax}\left(\frac{Q^{(r)}\left(K^{(r)}\right)^T}{\sqrt{d_{head}}}\right),\qquad O^{(r)}=A^{(r)}V^{(r)}$$

这里 A 是 attention weights，O 是这个 head 为每个 token 取回的结果。每个 head 都有自己的 Q、K、V、score、softmax 和 Value 聚合；不是所有 head 共用一份 Q/K/V 后只换一个名字。

为什么分数仍要除以 √dhead？

F-2 中的缩放理由没有消失，只是单头的 key 维度现在写作 d_head。每个 head 的 score 都先除以 √dhead，再做自己的 softmax。

为什么不只保留一个 head？

一个 head 当然可以通信，但它只能在一套 Q/K 几何空间里给关系打分，也只能在一套 V 空间里取回内容。多个独立 head 给模型多个可学习的投影空间：同一个输入位置可以在不同空间里形成不同的匹配模式和内容读取方式。

这不是“head 1 天生找主语，head 2 天生找宾语”。训练开始时所有矩阵通常随机；Loss 通过反向传播分别调整它们。一些训练后的 head 可能出现不同偏好，也可能重叠、难以解释，或在某些输入上几乎不用。结构只提供了不同的参数通道，不保证人工可命名的分工。

教学中的 head A 可以把“她”更多地对准“小红”，演示一条指代相关的匹配模式。
教学中的 head B 可以把同一个“她”更多地对准“指”，演示另一条关系模式。

上面只是帮助观察“同一 Query 能得到两套权重”的 toy 例子，不是在给真实模型里的 head 贴固定语义标签。

concat：不是拼 token，也不是平均

假设每个 head 都为第 i 个 token 输出一小段向量 o_i^(r)。concat 做的是把同一个 token在不同 head 的结果沿最后一个 feature 维排在一起：

$$c_i=[o_i^{(1)}\,\Vert\,o_i^{(2)}\,\Vert\,\cdots\,\Vert\,o_i^{(h)}]$$

如果一个 token 在两个 head 的输出分别是 [2, 5] 和 [7, 1]，concat 得到 [2, 5, 7, 1]。它不是拼 token，所以 token 数 n 不变；它不是平均，所以也没有把两条信息压成一个数。只是最后一个 feature 维变宽了。

token “她”[2, 5]||[7, 1]→[2, 5, 7, 1]

head 的排列顺序本身没有“第一个更重要”的人类语义；后面的 W_O 会学习怎样读取和混合这个拼接向量。

WO：不是额外一次 Attention，而是输出投影

拼接后的向量已经带着多个 head 的结果，但它还不是 Transformer residual stream 的标准宽度。输出矩阵 W_O 对每个 token 的拼接向量做一次普通 Linear：

$$Y=\operatorname{Concat}(O^{(1)},\ldots,O^{(h)})W_O+b_O$$

在常见行向量约定下，W_O 的 shape 是 [h d_head,d_model]。它把 h d_head 个拼接 feature 映射回 $d_{model}$ 个输出 feature，因此 Y 可以和原本的 residual stream 相加。

W_O 不是另一次“谁看谁”的 Attention，也不是纯粹的 reshape。reshape 只是换张量的看法，不改变数值；W_O 是可训练的线性变换，真的会学习各个 head 的 feature 应怎样组合。

把整条形状链路写成一份账本

下面采用最常见的等分设计：d_model = h d_head。例如 512 = 8 × 64。为了表达 batch 维，我们用 B；序列长度仍用 n。

输入 X[B,n,d_{model}]每个 batch 的每个 token 有 dmodel 个 feature。
Q / K / V[B,h,n,d_{head}]每个 head 各有一小段 Q、K、V。
scores / weights[B,h,n,n]每个 head 内，每个 Query 对所有 Key 的分数或权重。
每 head 输出 O[B,h,n,d_{head}]每个 token 在每个 head 内取回 dhead 维内容。
concat[B,n,h d_{head}]只把最后一个 feature 维拼宽，n 不变。
输出 Y[B,n,d_{model}]经过 WO 回到 residual stream 所需的宽度。

等分只是常见选择。更一般地，concat 的宽度是 h d_head，W_O 负责把它映射回 d_model。教材先用等分，是为了让形状最容易追踪。

从 GPU 看：不是 h 个 Python for 循环

概念上，我们把每个 head 写成独立的 W_Q^(r)、W_K^(r)、W_V^(r)。实现上常把同类矩阵沿输出 feature 拼成一张大矩阵，一次大矩阵乘法得到所有 head 的 Q，再 reshape 和 transpose 出 head 轴：

逻辑上的多头，GPU 上的打包张量
X: [B,n,d_model]
Q_big = X @ W_Q
Q: [B,h,n,d_head]

scores = Q @ K.transpose(-2,-1)
weights = softmax(scores, key axis)
O = weights @ V

concat: [B,n,h*d_head]
Y = concat @ W_O

这里的 reshape 和 transpose 只是把同一批数组织成方便批量矩阵乘法的视图；它们不让 head 变成 token，也不让 head 变成 Transformer 层。GPU 可以在 batch、head、token 等维度上并行处理，但每个 head 仍使用不同的参数切片。

mask 仍在每个 head 的 softmax 前

因果 mask、padding mask 或模态 mask 都应加在 score 上、softmax 前。对每个 head 的每一行 Query 来说，被遮住的 key 得到极小分数，softmax 后权重接近 0，所以它的 Value 不会被取回。

$$A^{(r)}=\operatorname{softmax}\left(\frac{Q^{(r)}\left(K^{(r)}\right)^T}{\sqrt{d_{head}}}+M\right)$$

工程中一个形状为 [B,1,n,n] 的 mask 可以广播给所有 h 个 head；这表示“每个 head 看同一组允许/禁止位置”。更复杂的架构也可以设计不同规则，但本节的标准 self-attention 假设同一 mask 作用于每个 head。

本节只讲 self-attention：Q、K、V 都来自同一个 X。cross-attention 中 Query 的长度可以与 Key/Value 的长度不同，但 head、concat 和 W_O 的基本思想仍然相同。

训练如何让不同 head 出现不同偏好？

训练开始时，所有 head 的参数通常是随机的。模型没有收到“你负责代词”“你负责语法”的人工标签；最终任务的 Loss 通过反向传播改变 W_Q、W_K、W_V、W_O 等全部相关参数。

如果某种匹配和内容读取有助于降低 Loss，梯度会逐步让对应 head 的参数朝有利方向更新。不同初始化、数据和优化过程可能让多个 head 学出不同模式，也可能没有清晰分工。没有任何结构保证每个 head 都重要、可解释或互不重复；训练不保证每个 head 都有一个可命名的专属职责。

标准 MHA 之外

MQA 和 GQA 会让多个 Query head 共享一部分 K/V head，以减少推理时 KV cache 的成本。它们是常见变体，但不改变这里先建立的核心：head 是一条投影与 attention 计算通道，不是一个 token。

小检查：现在能分清这些说法了吗？

  1. “一个 head 就是一个 token。”不对。head 是同一层 Attention 的一套参数和计算；每个 head 都处理所有 token。
  2. “多个 head 只是把 token 分组。”不对。每个 head 都看完整可见序列，只是用不同 Q/K/V 投影。
  3. “softmax 在所有 head 之间挑一个最重要的 head。”不对。softmax 在每个 head 内、沿 Key token 轴把 score 变成权重。
  4. “concat 是把更多 token 接到序列后面。”不对。concat 沿 feature 维拼接同一 token 的多条输出，序列长度不变。
  5. “W_O 只是改 shape。”不对。W_O 是可训练 Linear，用来混合 concat 后的 feature 并回到 dmodel。

带走这条多头主线

F-2 推导了一个单头 Attention：按 Q/K 匹配，用 softmax 得权重，再混合 Value。Multi-Head Attention 让同一个输入并行经过多套这样的计算；把各 head 输出 concat，再经 WO 回到 d_model。下一节 F-2.5 解释 MLP 如何对每个位置做非线性计算，之后 F-3 再把多头 Attention、Residual、LayerNorm 和 MLP 组装成完整 Transformer Block。
