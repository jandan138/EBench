Multi-Head Attention 从零开始（head / concat / WO）

F-2 把一套 Q/K/V 算到了加权 Value。那不是“Attention 的一小部分”，而是一条完整的 Attention 支路，也就是一个 head。现在先走完一条连续故事：一个 head 怎样变成多个 head，多个结果怎样 concat，最后的 WO 又在做什么。形状、GPU、mask 和 MQA/GQA 放到第二遍再看。

先给总画面，也先把符号翻译成人话

Multi-Head Attention 的输入仍是一批当前 hidden states。下面图里的 shape 先只当作标签；不必一开始就会矩阵乘法。

B一次一起算多少条序列，叫 batch size。
n一条序列有多少个 token 位置。
dmodel代码里常写作 d_model：每个 token 当前向量有多少个 feature，也叫模型宽度。
h / dheadh 是并行支路数；每条支路内部使用 d_head 个 feature。

把它看成：同一个输入 X，同时交给 h 套不同参数的单头 Attention。每一套都完成 F-2 的完整链路。直到各自把 Value 混合完，结果才被拼在一起。

同一个输入 X [B,n,dmodel] 每个位置当前的 hidden state
→
head 1 WQ(1), WK(1), WV(1) 一套完整单头 Attention
head 2 WQ(2), WK(2), WV(2) 另一套完整单头 Attention
→
concat [B,n,h dhead] 拼同一 token 的 feature
→
Linear WO [B,n,dmodel] 混合多条支路的结果
图里的“并行”不是多个小人

这是讲数据流的方便说法。真实实现保存的是张量和参数矩阵；GPU 能沿 head 这一维批量计算。head 没有自己的存储、人格或函数，它是一套参数切片和相应的计算通道。

head 到底在哪条坐标轴上？

“第 3 个 token”“第 3 层”“第 3 个 head”听起来相似，却是三个不同问题：

token / 位置序列里的第几个位置，例如“她”或“小红”。有 n 个。
feature / 通道一个位置的向量里第几个数。有 dmodel 个。
Transformer 层第几次完整更新 hidden state。每层有自己的参数。
head同一层 Attention 内的一套 Q/K/V 投影与计算。有 h 个。

一个 head 不是 token，不是位置，也不是 Transformer 层。它处理所有可见 token，只是使用自己的投影参数。

先站稳：一个 head 本身已经是完整 Attention

令第 r 个 head 的参数带上上标 (r)。它拿到的输入仍是同一个当前 hidden-state 张量 X，只是用自己的三套矩阵投影：

$$Q^{(r)}=XW_Q^{(r)},\qquad K^{(r)}=XW_K^{(r)},\qquad V^{(r)}=XW_V^{(r)}$$

接着完全照着 F-2 的单头流程走：

$$A^{(r)}=\operatorname{softmax}\left(\frac{Q^{(r)}(K^{(r)})^T}{\sqrt{d_{head}}}\right),\qquad O^{(r)}=A^{(r)}V^{(r)}$$

A 是这个 head 内每个 Query 对 Key token 的权重，O 是它为每个 token 取回的结果。标准 MHA 中，每个 head 有自己的 Q、K、V、score、softmax 和 Value 聚合；不是共用一份 Q/K/V 后只换名字。

为什么不只保留一个 head？

一个 head 当然可以通信，但它只能在一套 Q/K 几何空间里给关系打分，也只能在一套 V 空间里读取内容。多个独立 head 给模型多个可学习的投影空间：同一个位置可以在不同空间里形成不同的匹配模式和内容读取方式。

这不是“head 1 天生找主语，head 2 天生找宾语”。训练开始时参数通常随机；Loss 通过反向传播分别调整它们。一些 head 可能形成不同偏好，也可能重叠、难以解释，或在某些输入上几乎不用。结构提供不同参数通道，不保证人工可命名的分工。

教学中的 head A可以让“她”更多地对准“小红”，演示一条指代相关的匹配模式。
教学中的 head B可以让同一个“她”更多地对准“指”，演示另一条关系模式。

这是帮助观察“同一 Query 可得到两套权重”的 toy 例子，不是在给真实模型里的 head 贴固定语义标签。

concat：不是拼 token，也不是平均

假设每个 head 都为第 i 个 token 输出一小段向量 o_i^(r)。concat 把同一个 token在不同 head 的结果沿最后一个 feature 维排在一起：

$$c_i=[o_i^{(1)}\,\Vert\,o_i^{(2)}\,\Vert\,\cdots\,\Vert\,o_i^{(h)}]$$

如果一个 token 在两个 head 的输出分别是 [2, 5] 和 [7, 1]，concat 得到 [2, 5, 7, 1]。它不是拼 token，所以 token 数 n 不变；它不是平均，所以没有把多条信息压成一个数。只是这个 token 的 feature 向量变宽了。

token “她”[2, 5]||[7, 1]→[2, 5, 7, 1]

WO：把拼接结果做一次普通 Linear

concat 后的向量已经带着多个 head 的结果。WO 不再做“谁看谁”的匹配；它是一次普通 Linear，也就是对 feature 做可学习的加权组合。令 C 表示 concat 后的张量：

$$Y=CW_O+b_O$$

对一个 token 的一个输出 feature 来说，就是“C 里的每个数乘一个权重、全部相加，再加上一个 bias”。例如 c=[2,5,7,1] 时，某一维可以是 y=2w_1+5w_2+7w_3+1w_4+b。这就是加权组合；b_O 是可学习的偏置，有些具体模型会省略它。

常见行向量约定下，W_O 的 shape 是 [h d_head,d_model]。它把 concat 的 h d_head 个 feature 映射回 d_model 个 feature。这个“回到原宽度”是后续 Transformer Block 能做逐元素相加的形状前提；residual 的意义会在 F-3 再定义。

W_O 不是另一次 Attention，也不是纯 reshape。reshape 只改变怎么看同一批数；W_O 会改变数值，并学习怎样混合多个 head 的 feature。

第一遍走完：多头到底多了什么？

  1. 每个 head 都是完整的单头 Attention，不是一个 token。
  2. 多个 head 用不同参数并行处理同一个输入 X。
  3. concat 只把同一 token 的 feature 拼宽，不改变 token 数。
  4. WO 是普通 Linear，把拼接向量混合并映射回 dmodel。
  5. 若有 causal 或 padding mask，它仍应在每个 head 的 softmax 前作用在 score 上。

第二遍再看：形状、GPU 与边界

这一组内容是可见的进阶补充，不会藏进折叠面板；从搜索或书签跳到其中任一小节也能单独阅读。

把整条形状链路写成账本

采用最常见的等分设计：d_model = h d_head，例如 512 = 8 × 64。B 是 batch size，n 是序列长度。

输入 X[B,n,d_{model}]每个 batch 的每个 token 有 dmodel 个 feature。
Q / K / V[B,h,n,d_{head}]每个 head 各有一小段 Q、K、V。
scores / weights[B,h,n,n]每个 head 内，每个 Query 对所有 Key 的分数或权重。
每 head 输出 O[B,h,n,d_{head}]每个 token 在每个 head 内取回 dhead 维内容。
concat[B,n,h d_{head}]只把最后一个 feature 维拼宽，n 不变。
输出 Y[B,n,d_{model}]经过 WO 回到标准模型宽度。

从 GPU 看：不是 h 个 Python for 循环

概念上可以把每个 head 写成独立的 W_Q^(r)、W_K^(r)、W_V^(r)。实现上常把同类矩阵沿输出 feature 拼成一张大矩阵，一次大矩阵乘法得到所有 head 的 Q，再 reshape 和 transpose 出 head 轴：

逻辑上的多头，GPU 上的打包张量
X: [B,n,d_model]
Q_big = X @ W_Q
Q: [B,h,n,d_head]

scores = Q @ K.transpose(-2,-1)
weights = softmax(scores, key axis)
O = weights @ V

concat: [B,n,h*d_head]
Y = concat @ W_O

reshape 和 transpose 只是组织同一批数以方便大矩阵运算；它们不会让 head 变成 token 或 Transformer 层。

mask 仍在每个 head 的 softmax 前

因果 mask、padding mask 或模态 mask 都应加在 score 上、softmax 前。对每个 head 的每一行 Query 来说，被遮住的 key 得到极小分数，softmax 后权重接近 0，因此它的 Value 不会被取回。

$$A^{(r)}=\operatorname{softmax}\left(\frac{Q^{(r)}(K^{(r)})^T}{\sqrt{d_{head}}}+M\right)$$

工程中一个形状为 [B,1,n,n] 的 mask 可以广播给所有 h 个 head。全遮住一整行时，具体实现需要额外约定以避免 softmax 对全极小值产生无意义分布；标准 causal self-attention 不应制造这种可见 Query 行。

训练如何让不同 head 出现不同偏好？

训练开始时，所有 head 的参数通常是随机的。模型没有收到“你负责代词”“你负责语法”的人工标签；最终任务的 Loss 通过反向传播改变 W_Q、W_K、W_V、W_O 等参数。

如果某种匹配和内容读取有助于降低 Loss，梯度会逐步让对应参数朝有利方向更新。不同初始化、数据和优化过程可能让多个 head 学出不同模式，也可能没有清晰分工。没有结构保证每个 head 都重要、可解释或互不重复；训练不保证每个 head 都能得到一个人类可命名的职责。

标准 MHA 之外：MQA 与 GQA

上面“每个 head 独立 Q/K/V”说的是标准 MHA。MQA 让多个 Query head 共享一组 K/V；GQA 让若干 Query head 共享一组 K/V group。它们主要为了减少推理时 KV cache 的成本。核心直觉仍不变：Query head 仍是 Attention 的计算通道，不是一个 token。

小检查：现在能分清这些说法了吗？

  1. “一个 head 就是一个 token。”不对。head 是同一层 Attention 的一套参数和计算；每个 head 都处理所有 token。
  2. “多个 head 只是把 token 分组。”不对。每个 head 都看完整可见序列，只是用不同 Q/K/V 投影。
  3. “softmax 在所有 head 之间挑一个最重要的 head。”不对。softmax 在每个 head 内、沿 Key token 轴把 score 变成权重。
  4. “concat 是把更多 token 接到序列后面。”不对。它沿 feature 维拼接同一 token 的多条输出。
  5. “WO 只是改 shape。”不对。WO 是可训练 Linear，用来混合 concat 后的 feature 并回到 dmodel。

带走这条多头主线

F-2 推导了一个单头 Attention：按 Q/K 匹配，用 softmax 得权重，再混合 Value。Multi-Head Attention 让同一个输入并行经过多套这样的计算；把各 head 输出 concat，再经 WO 回到 dmodel。下一节 F-2.5 解释同一位置的向量怎样进入 MLP；随后 F-2.6 再深挖 GELU 的门控；最后在 F-3 组装成完整 Block。
