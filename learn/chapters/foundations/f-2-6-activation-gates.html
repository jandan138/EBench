ReLU、GELU 与门控从零开始

F-2.5 已经让一条 hidden state 走完了 Linear 1 → GELU → Linear 2。这里再回到中间那一小步：Linear 刚输出的 z 到底是什么？为什么 ReLU / GELU 会对正负数不对称？训练又怎样学会把某些输入推向更容易通过的区域？

先放好四个容易混掉的东西

不要说“token 被推成正数”。通常被送进激活函数的是某个中间 feature 的数值：

token / 字符串文本里的离散符号或位置，不是一个实数。
hidden state x该位置当前的一整条向量，来自 embedding 或前面层的更新。
pre-activation zLinear 为某个中间通道算出的一个 affine score。
activation a把 z 送进 ReLU 或 GELU 后得到的数。

本章的因果链是：人为选一个正负不对称的非线性函数 → 训练根据 Loss 调整 W 和 b → 某个通道逐渐让自己关心的输入落到更容易通过的区域。不是“正数天生有意义，负数天生没有信息”。

先看激活前：z 是什么

F-2.5 用行向量写过 xW+b：一整条输入向量一次算出许多输出 feature。现在若只盯着其中一个输出通道，也可以把那一列权重写成向量 w，得到 w^T x+b。这是同一件事的两种排版约定，不是两个不同模块。

因此一个通道的 pre-activation 是 $z=w^Tx+b$。它叫“响应”或 score，只是因为它衡量当前输入在该学习到的方向上的 affine 投影；它不是一个已被证明对应“猫”“宾语”或“有用”的自然语言标签。

用冻结算例 CHANNEL_INPUT=[1,1]、CHANNEL_W=[[1,-1,0.5,-2],[1.1,0.3,0.8,-1.2]]、CHANNEL_B=[0,0,0,0]。第一个输出通道的逐项计算是 z_1=1(1)+1(1.1)+0=2.1。四个通道由同样的 affine 规则得到 [2.1,-0.7,1.3,-3.2]。

“response”不是小人真的回应问题

它只是“某个可学习函数在当前输入上给出的数值”。一个真实模型通常用分布式表示：一个维度很少能被可靠地命名成人类概念。

正负号不是“有用”与“无用”

z 的正负只表示它位于零点的哪一侧，也就是相对于当前学习到的方向是正投影还是负投影。它不天然表示“正确/错误”“存在/不存在”“好/坏”。换一套 feature 坐标，或让后层一起做匹配变换，解释都可能改变。

确实可以把权重方向反过来：把 w 换成 -w 会让投影符号翻转。正半轴被设计成“容易通过”只是约定；训练会适应这条约定，学会把它想让该通道响应的模式映到那一侧。

ReLU：一条明确的硬门

$ReLU(z)=max(0,z)$。当 z 严格为负时，输出为 0；当 z 为正时，输出就是 z；在 z=0 处函数连续但不可微，实现会选择一个 subgradient。它不是发现了“负数没有价值”，而是人为规定输入空间的两边采用不同规则。

这个设计有三个可分开的作用：

  1. 非线性：两层 Linear 不再能普遍合并成一层。
  2. 门控：某些通道在当前输入上完全关掉，另一些通过。
  3. 稀疏性：一批输入中一部分输出恰为 0；这可能有用，也可能让单元长期不激活而形成 dead ReLU。

也可以设计“保留负数、抑制正数”的镜像门。对前一层配合改变权重方向后，它与普通 ReLU 可以表达等价的函数族。重点在不对称的非线性，不在正号本身有特殊道德地位。

bias 怎样移动实际门槛

ReLU 的函数拐点固定在 z=0，但 z 本身是 $wx+b$。所以 bias 可以把输入空间中的实际门槛移动到别处。最简单的一维例子是 $ReLU(x-3)$：x 为 2、3、4 时，输出依次是 0、0、1。

高维时边界是 $w^Tx+b=0$。w 决定边界朝向，b 决定位置。w=0 时，z 完全由 b 决定；负 w 会反转一维不等式的方向。这些都是“门在哪儿”的几何问题，不是给一个 token 安装阈值属性。

GELU：不是硬砍，而是平滑的软门

GELU 也总体偏向让较大的正输入通过、压低较大的负输入，但不像 ReLU 那样一刀切成 0。先看数值：

optional 数学补充：概率式写法

精确定义是 $GELU(x)=x\Phi(x)$，其中 $\Phi$ 是标准正态分布的累积分布函数。若以概率 $\Phi(x)$ 保留 x，期望输出就是 $x\Phi(x)$。这是解释公式的随机门期望，不表示运行时真的为每个 feature 抽样 Bernoulli。

常见实现会用 tanh 近似；它近似的是同一个 GELU。较大负输入被压向 0，但不会像 ReLU 一样严格清零。

负数信息不等于被永久删除

ReLU 的确把这个通道的负 z 变成 0，但网络可以安排别的通道编码相反方向。令 $z=w^Tx+b$；相反分支用 -w 和 -b，得到 -z。下游再把两条 ReLU 输出相减，就有：

$$ReLU(z)-ReLU(-z)=z$$

这不是说每个真实网络都会严格配对，只说明“一个 ReLU 把负半轴清零”不能推出“相反方向的信息在整个网络里不可表达”。

GELU 更直接：$GELU(-1)\approx-0.159$。它若再乘后层的负权重 -2，会给后续和带来正贡献约 0.318。负激活和负权重的组合不等于删除信息。至于 Transformer Block 怎样通过 residual 传递原表示，留给 F-3 的 Residual。

ReLU(z)-ReLU(-z)=z；z=-2.500 时为 0.000-2.500=-2.500。

训练怎样利用这条不对称规则

可以这样理解：激活函数先规定“较正的 z 比较容易通过”；训练再根据最终任务的 Loss 调整 W 和 b，让某个通道在有助于任务的输入上更可能落到该区域，在不相关输入上较少强响应。“有助于”由数据、任务头和 Loss 间接决定，不是人手给某个通道贴“猫”或“宾语”的标签。

addendum：SiLU 与 SwiGLU 为什么出现

ReLU 与 GELU 都是「一条规则管一个数」的逐元素门。后来者们只做了一点变化：SiLU（又叫 Swish）是 $x\cdot\operatorname{sigmoid}(x)$——用 x 自己当门控信号，形状与 GELU 很接近，同样平滑、保留少量负值，但计算更简单。既然激活函数的选择标准是「非线性 + 平滑门 + 好优化」，而不是某个唯一正确形状，工程上就会在这族近亲里挑顺手的一个。

SwiGLU 走得更远一步：它不再逐元素，而是把 MLP 的第一层 Linear 拆成两支，一支过 SiLU 当「阀门」，另一支不过激活当「内容」，两支逐元素相乘后再进 Linear 2。直觉：原来「门」的形状是激活函数固定的，现在门本身也带可学参数——网络可以自己学出对每个 feature「放多少过去」。代价是多一组权重，收益是表达力与训练的稳定提升，所以 LLaMA、Qwen 这一代模型普遍用它替换 GELU MLP（F-3 的变体表里那一行，说的就是这件事）。

检查：把这条因果链说准确

  1. z 的正负不是天然的“有用/无用”标签，而是相对于学习到的方向位于零点哪侧。
  2. ReLU / GELU 的关键是对不同区域使用不同规则，从而提供非线性和门控。
  3. 正半轴是设计约定；训练会调整 W、b，使某个通道把对任务有帮助的模式映到更容易通过的区域。
  4. bias 改变 z=0 在输入空间里的位置，因此能移动实际阈值。
  5. ReLU 清零一个通道的负响应，不等于整个网络永久失去相反方向的可表达性。
  6. GELU 通常保留少量负值；负值随后乘负权重仍可给后续结果正贡献。

带着门控直觉进入 Transformer Block

现在可以把 MLP 中间步骤说清楚：Linear 先产生 pre-activation z；激活函数按预先规定的非线性规则把 z 变成 a；Loss 再经反向传播调整 W、b，让不同通道逐渐在不同输入区域响应。回到 F-2.5 的完整 MLP trace 后，继续读 F-3 的完整 Block 数据流；LayerNorm 会在 F-3 LayerNorm 中单独展开。
