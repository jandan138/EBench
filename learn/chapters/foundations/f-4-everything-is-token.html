一切皆 Token + Embedding

Transformer 眼里只有一件事：一排 token。图像、语言、机器人状态、动作，最后都要变成这排向量里的成员。这就是“一切皆 Token”的世界观。

Token 到底是什么

Token 就是一个可查表的编号 + 对应的向量。模型先把输入拆成编号（tokenization），再通过 Embedding 表查出对应的向量。

  • 文字 → BPE / WordPiece token
  • 图像 → patch 或视觉 token
  • 机器人状态 → 直接线性投影成向量
  • 动作 → 离散化后成 token，或直接作为连续向量参与 flow

Embedding 的作用

Embedding 就是一个巨大的查表矩阵。输入一个 token id，输出一个固定维度的向量。这个向量一开始是随机初始化的，通过训练学会“相近意思的词/状态/动作应该有相近的向量”。

对 VLA 的意义

当我们把多相机图像、关节角、语言指令、action chunk 全部变成 token 序列后，同一个 Transformer 就可以用同一套机制处理它们。这也是为什么 π0、X-VLA、InternVLA 都长得像“多模态语言模型 + 动作头”。

这一节你需要带走的

Transformer 不认图像、不认状态、不认动作，它只认 token 序列。Embedding 把“东西”变成向量。后面所有多模态融合、action chunk、FAST 离散化，都是在这个世界观下发生的。
