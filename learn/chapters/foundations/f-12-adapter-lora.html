Adapter 与 LoRA：冻结 backbone 时梯度怎么走

这一节是进阶可选的一课：它回答「backbone 已经是别人训好的，能不能只训一小部分参数就让它输出动作」。主线一句话：冻结不是断路——前向照走、梯度照穿，只是 optimizer 不更新被冻的参数；Adapter 和 LoRA，就是在 frozen 主干旁开的两条「可学旁路」。读完它，你再看 5b-5 的 KI 和 6-2 的 X-VLA soft prompt，会有「原来都是一家人」的感觉。

前置与定位

需要 f-10 的梯度与参数清单。本书的 baseline 并没有直接使用 Adapter/LoRA（π0.5 用 KI、X-VLA 用 soft prompt），本节是理解它们的旁路视角。

为什么要冻结：三笔账

全量微调一个 3B 模型意味着：所有参数都要存梯度、存 optimizer 状态（Adam 还要再存两份动量），显存翻好几倍；更重要的是第二笔账——灾难性遗忘：为机器人数据改动全部参数，会把预训练时学到的语言与视觉知识一起冲掉（5b-5 把这叫「污染」）。第三笔账是复用：一份主干服务一百个任务/机器人，总不能存一百份完整模型。于是思路很自然：主干冻结，只在旁边加少量可学参数，每个任务只保存自己那一小份。这就是 PEFT（parameter-efficient fine-tuning）。

Adapter：插在层间的瓶颈小网络

Adapter 的结构一句话说完：降维 → 激活 → 升维。在 Transformer 子层之后插一个小 MLP：先把 d 维压到很窄（如 64 维，这道窄口叫 bottleneck），过一次非线性，再升回 d 维，作为修改量加回去：

$$\text{output} = F(x) + \text{Adapter}(x),\qquad \text{Adapter}(x)=\sigma(xW_{\text{down}})\,W_{\text{up}}$$

冻结主干后，每个 Block 里只有这对小矩阵可学——参数量常常不到主干的 1%。直觉：主干已经把表示加工得八九不离十，Adapter 只需学「针对这个新任务，往哪个方向轻轻推一把」。瓶颈保证它只能学低成本的修改，没法另起炉灶。

LoRA：并在线性层旁的低秩修改

LoRA 的思路更巧妙：不新增模块，而是给已有的线性层加一个并联修改量。冻结原始权重 W，学一个低秩矩阵 $\Delta W = BA$（B 是 d×r，A 是 r×d，秩 r 常取 8~64），前向变成：

$$y = (W + \Delta W)\,x = Wx + BAx$$

为什么低秩就够？经验事实：把一个通用模型适配到具体任务，权重的有效修改量往往远低于全秩——任务不需要重写 W，只需要在某些方向上做有限调整。r 就是容量旋钮：r 越小越省、越不容易过拟合，越大越接近全量微调的表达力。部署时还能把 BA 合并进 W，推理零额外开销。可以把它想成：Adapter 是「在流水线中间加一道小工序」，LoRA 是「给机床本身贴一张可换的校准片」。

冻结时，loss 的梯度到底怎么走

最常见的误解是「冻结 = 这部分被切出计算图」。不对。冻结层前向照常参与计算，反向时梯度照常穿过它——只有穿过它，梯度才能到达更上游的 LoRA/Adapter 参数；区别只在最后一步：optimizer 更新参数时，跳过所有被标记为 frozen 的参数。所以完整路径是：

冻结主干时的一步训练
前向：x → frozen backbone → 可学旁路/head → loss
反向：loss 的梯度穿过 head → 穿过 backbone（照穿，不更新）
      → 抵达 LoRA 的 B、A / Adapter 的两个 W / head 的权重
更新：只有这些可学参数被 optimizer 改动

还要和另一种机制分清：冻结是「梯度穿透但不更新」，stop-gradient（截断）是「梯度根本过不去」。VLA 里的 knowledge insulation 属于后者——它故意把 flow matching loss 的梯度挡在 backbone 门外，同时让别的 loss（CE）照常更新 backbone。一张对照表见 5b-5 的「KI ≠ 冻结」：两者一个是「不更新」，一个是「不让传」，别混。

interactive三种更新制度：谁亮谁冻
切换 全量微调 / Adapter / LoRA：亮起的模块会被 optimizer 更新；灰色模块前向照走、梯度照穿，只是不被更新。
从 RL 看 冻结主干 ≈ 固定特征，只训 head

RL 里「固定 pretrained 特征提取器、只训 actor/critic 头」的做法和 PEFT 同宗；stop-gradient 则是 target network 思想的另一种形态——都是「让某些梯度别乱传，稳住表示」。差别只在粒度：PEFT 在主干内部埋可学旁路，比「只训头」表达力高一个档。

把地图拼起来：全量 / PEFT / KI

现在可以给「怎么改别人的模型」画一张三分地图：

  • 全量微调：全部参数可学。表达力最高，最贵，遗忘风险最大。
  • PEFT（冻结主干 + 可学旁路）：Adapter（层间瓶颈）、LoRA（层旁低秩）、soft prompt（输入端可学 token，6-2 的 X-VLA 用它，~9M 参数接一个新机器人）。梯度穿透主干但不更新它——「硬冻结」。
  • KI（stop-gradient）：主干不冻结，仍被 CE 类 loss 更新；只是连续动作 loss 的梯度被截断——「软隔离」。详见 5b-5。

最后把 f-9 的主线图补全成训练版，作为 Part F 的收束：

全链主线（训练版）
Image + Language + State → Embedding → Transformer Blocks → H [B,n,d]
  → 任务头按读法取行（f-9 / f-11）→ 预测 → loss（f-5 / f-6）
  → backprop 沿高速公路回传（f-10）
  → 更新：全量 / 只更新旁路与头（Adapter · LoRA · soft prompt）
     / 或截断特定 loss（KI）

检查：把这几句话说准确

  1. 「冻结的层不参与计算。」不准确：前向照走、梯度照穿，只是 optimizer 不更新它。
  2. 「Adapter 输出替换原层输出。」不准确：它是加在原输出上的修改量（降维→激活→升维的小网络）。
  3. 「LoRA 让模型变小。」不准确：前向是 (W+BA)x；部署可合并，训练时省的是可学参数与 optimizer 状态。
  4. 「KI 是一种特殊的冻结。」不准确：KI 用 stop-gradient 截断特定 loss 的梯度，主干仍被别的 loss 更新。

这一节你需要带走的

冻结的动机是显存、遗忘与复用；Adapter 是层间瓶颈小网络，LoRA 是层旁低秩 ΔW=BA；冻结时梯度穿透主干、只更新旁路与头，stop-gradient 则直接截断。至此 Part F 的 12 节闭环：从 token 到 hidden state（f-1~f-4）、从 loss 到训练回路（f-5~f-10）、从读取到微调（f-11~f-12）。带着这张地图进入 Part 0，后面每个概念都有地方落脚。
