从 RL 到 behavior cloning：生成式策略

你带着 RL 的工具箱走进这本书：policy、value、reward、policy gradient、TD。这一节我们从这套你最熟的语言出发，看清一件事——为什么 manipulation 这个领域，主流方法不是你预期的 online RL，而是 imitation / behavior cloning，以及由此长出的「生成式策略」到底新在哪。

先把 RL 摆在桌上

RL 的世界你很熟：一个 agent 在 environment 里，policy $\pi(a\mid s)$ 决定在状态 $s$ 下取动作 $a$，环境回一个 reward $r$，目标是最大化期望累积回报 $\mathbb{E}\big[\sum_t \gamma^t r_t\big]$。你有两条主路去优化它：policy gradient（直接对 $\pi$ 求梯度，如 REINFORCE / PPO）和 value-based（学 $Q(s,a)$ 或 $V(s)$，用 TD 误差自举，再 $\arg\max_a Q$ 取动作）。无论哪条路，前提都是：你能大量、廉价、安全地和环境交互，并拿到有意义的 reward 信号。

从 RL 看 三个前提，在 manipulation 里全塌了

online RL 默认你能随便 rollout、reward 现成、试错没代价。但真实/仿真的灵巧操作里：rollout 不廉价（每步要推进一整个 Isaac Sim 物理世界）、reward 难定义（「把杯子稳稳放进杯托」这种目标很难写成稠密标量）、试错 不安全也不便宜（真机会撞坏，仿真要烧 GPU 时）。三根支柱一起松动，online RL 的性价比就崩了。

为什么 manipulation 转向 behavior cloning

既然在线试错这么贵，一个更省的思路是：别让机器人自己摸索，直接给它看专家怎么做。这就是 imitation learning 里最直接的一支——behavior cloning（BC）。我们收集一批专家演示 $\{(o_i, a_i)\}$（人类遥操作、或 rule-based 脚本生成的轨迹），然后纯监督学习地拟合一个策略 $\pi_\theta(a\mid o)$，让它在见过的观测上复现专家动作。

没有 reward、没有 critic、没有 Bellman 方程、没有环境交互——训练时只有一堆 (观测, 动作) 对和一个回归/似然损失。EBench 的训练数据正是为此而生：EBench-Dataset 的 6,600 条轨迹里，既有 rule-based（GenManip 脚本）生成的 long_horizon / simple_pnp，也有人类 teleop 的轨迹（2.4 细讲）。

BC 不是免费午餐

BC 有个著名的软肋：compounding error / covariate shift——模型只在专家走过的状态上训练，一旦推理时犯一点小错偏离了专家分布，就进入没见过的状态，错误像滚雪球一样累积。这个问题怎么缓解（答案是 action chunking），我们留到 5.1 专门讲。这里先记住它存在。

关键转折：动作分布是多模态的

现在到了这一节真正的核心。你可能会想：BC 不就是个回归吗，$\pi_\theta(o)\to a$，最小化 $\lVert \pi_\theta(o)-a\rVert^2$ 就完了？问题恰恰出在这里。

设想一个场景：桌上一个瓶子，专家演示里，有人从左边绕过去抓、有人从右边抓——两种都对。你的数据里同一个观测 $o$ 对应了两个差异很大的「正确」动作。如果你用均方误差去回归，模型会学到什么？它会学到两个动作的平均——一个直直撞向瓶子、两边都不沾的废动作。这就是 mode averaging（模式平均）的灾难。

问题的根子是：从观测到动作的映射不是一个函数，而是一个分布——$p(a\mid o)$ 往往是multimodal（多模态）的，一个观测下有多种同样合理的做法。我们需要的策略，必须能表达并采样这个多模态分布，而不是塌缩到它的均值。

「生成式策略」是什么意思

于是有了这本书反复出现的关键词：generative policy（生成式策略）。它的定义很朴素——策略是一个生成模型，能从条件分布 $p(a\mid o)$ 里采样出动作，而不是输出一个确定的点。给定观测，它可以这次采样出「从左边抓」，下次采样出「从右边抓」，两次都合理，而不会平均成废动作。

把它和你脑中的 RL 策略对照一下，差异就清楚了：

从 RL 看 它最像 stochastic actor，但训练方式不同

生成式策略最接近的 RL 类比是一个 stochastic actor——一个会输出动作分布、可以从中采样的随机策略。但有两点关键不同：(1) 它不靠 policy gradient / TD 训练，而是用模仿（behavior cloning）的去噪 / 似然损失；(2) 它不是 argmax over $Q$——value-based 方法选「Q 值最高的那一个」动作，本质是单峰、确定的；生成式策略要的恰恰相反：保留并采样整个多模态分布。换句话说，没有 critic、没有 Bellman，只有「学会专家动作的分布」。

那「生成模型」具体用什么？早期 VLA 把动作离散成 token 自回归地吐（RT-2 / OpenVLA），后来 π0 这一路改用 flow matching——一种能在连续高维动作空间上高效采样多模态分布、且推理只需极少步的生成方法（1.2 先点名，5.2 钻数学）。它训练时本质就是 behavior cloning：从 $p(A_t\mid o_t)$ 采样，没有 Q、没有 critic。这正是我们为 Part 5 埋下的伏笔。

一句话拎清三个词的关系

imitation / behavior cloning 是训练范式（学专家，不试错）；multimodal action distribution 是问题性质（同一观测有多种正确做法）；generative policy 是应对方案（用生成模型采样分布，而非回归均值）。三者一环扣一环，构成了现代 VLA 的方法论底色。

这一节你需要带走的

manipulation 主流不用 online RL，是因为 rollout 贵、reward 难、试错危险；转而用 imitation / behavior cloning 从专家演示监督学习。但简单回归会因 multimodal action distribution 而 mode-average 出废动作，所以策略必须是一个能采样分布的 generative policy——它像个 stochastic actor，却用 BC 而非 policy gradient 训练，也不做 argmax over $Q$。
