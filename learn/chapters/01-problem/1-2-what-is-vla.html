什么是 VLA：VLM backbone + action

上一节我们确立了：操作策略要是一个能采样多模态分布的生成式策略。但策略得先看懂世界、听懂指令才谈得上动作。这一节回答 VLA 这个缩写到底是什么——以及它为什么要站在一个预训练 VLM 的肩膀上。

VLA = VLM 骨干 + 输出 action 的能力

VLA 是 Vision-Language-Action 的缩写。它的配方一句话就能说清：拿一个预训练好的 VLM（vision-language model）当骨干，再给它加上输出 robot action 的能力。视觉与语言这两个模态，VLM 已经处理得很好了；VLA 做的，是在这个强大的多模态理解之上，再接出一条通往「动作」的路。

所以一个 VLA 同时吃三种东西、吐一种东西：吃图像（Vision）、吃语言指令（Language），最后吐动作（Action）。前两个字母是继承来的家底，第三个字母是新长出来的能力。

为什么非要借一个 VLM

你可能会问：为什么不从零训练一个专用的操作网络，非要扛着一个几十亿参数的 VLM？答案是VLM 自带的语义、常识与泛化。一个在海量图文上预训练过的 VLM，已经知道「apple」长什么样、「左边」是哪边、「把它放进碗里」是什么意思——这些语言与视觉的世界知识，如果靠那点机器人演示数据从头学，根本学不出来。

从 RL 看 这是大规模 pretraining 当 prior

用 RL 的话说，VLM 骨干给了策略一个极强的 prior / 表示初始化——相当于先在一个巨大的、与任务无关的语料上做了表示学习，再用少量演示「微调」出动作。这让样本效率和泛化大幅提升：换一句没见过的指令、换一个没见过的物体，VLM 的语义先验能帮策略「推断」该怎么做，而不是只会复读训练集里的具体轨迹。

一个具体骨干：PaliGemma

给个具体的例子。π0 这一路 VLA 用的骨干是 PaliGemma——一个 3B 的 VLM。它由两块拼成：一个 SigLIP 视觉编码器（≈400M，负责把图像编码成视觉 token）加一个 Gemma 语言模型（≈2.6B，负责融合视觉与语言、做推理）。

数字的口径

π0 paper 本身只说骨干是「3B 的 PaliGemma」。SigLIP≈400M、Gemma≈2.6B 是背景知识（PaliGemma 的公开构成），帮你理解这 3B 是怎么分的，并非 paper 给出的精确切分。本书把这类背景知识标出来，免得你误以为是论文里的硬指标。

两条「长出 action」的路线

VLM 骨干有了，怎么从它身上接出动作？这里 VLA 分出两条历史路线，理解它们的差别，是看懂整个领域的关键。

路线一：离散 token + 自回归

早期 VLA（RT-2 / OpenVLA）的做法非常「语言模型」：把连续的动作离散化成 token，然后像生成文字一样，一个一个 token 自回归地吐出来。好处是几乎不用改 VLM 的架构——动作就是另一种「语言」，复用整套 next-token 预测。但代价是：自回归逐 token 解码慢，离散化也损失精度，对高频、灵巧的连续控制并不友好。

路线二：flow matching

π0 这一路主张：离散 token 自回归不适合高频灵巧控制，转而用 flow matching 直接在连续动作空间上生成。这正好接上 1.1 说的生成式策略——flow matching 能表达多模态连续动作分布，且因为走「近直线」的概率路径，推理只需极少步积分，适合实时控制。

点到为止

flow matching 的数学（概率路径、向量场、forward Euler 解 ODE、rectified flow 为何能少步）是 Part 5 的重头戏，5.2 会带你一步步推。这里你只要知道它是「连续、生成、可少步采样」的路线，和离散自回归并列即可。EBench 的几个 baseline（π0/π0.5/X-VLA/InternVLA-A1）都属于这一连续生成路线。

输入输出长什么形状

最后把一个 VLA 在 EBench 里的 I/O 形态落地。输入是多相机图像 + 语言指令 + proprioception（本体感知，如关节角），输出是一段 action chunk（一次预测未来若干步动作，而非单步——为什么这样做见 5.1）。具体到 EBench 本体 lift2：

角色内容来自
Vision多相机 RGB（lift2 有 4 个 480×640 视角）video.{cam}_view
Language自然语言任务指令instruction
Proprio关节角 / 夹爪 / 底盘 / ee_pose 等本体状态state.*
Action一段未来动作 chunk（关节 / 夹爪 / 底盘运动）模型输出

这套观测/动作的精确字段、维度与契约，是 Part 4 的内容；本体 lift2 的细节就在下一节。

这一节你需要带走的

VLA = 预训练 VLM 骨干 + 输出 action 的能力；借 VLM 是为了白拿它的语义、常识与泛化（一个强 prior）。π0 用 PaliGemma（3B = SigLIP + Gemma，后者为背景）。从骨干长出动作有两条路：早期 RT-2/OpenVLA 的离散 token 自回归，和 π0 这一路的 flow matching（连续、生成、可少步，细节留 Part 5）。I/O 形态：多相机图 + 语言 + proprio → action chunk。
