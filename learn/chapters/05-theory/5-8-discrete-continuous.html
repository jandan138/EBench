离散 vs 连续、FAST、knowledge insulation

π0 已经能在实验室里灵巧操作。但「进一个从没见过的家、把厨房收拾干净」是另一个量级的问题——这是 π0.5 要解的 open-world generalization。它逼出三个深刻问题：动作该用离散还是连续表示？高层任务规划和低层控制怎么共处？以及——怎么在学控制的同时不把 VLM 的世界知识冲掉？

π0.5：从实验室走向开放世界

π0.5（arXiv 2504.16054）建立在 π0 之上，目标是 open-world generalization：机器人要走进一个训练中从未出现过的真实家庭，完成清厨房、铺床、挂毛巾这类多阶段任务。这比固定实验室难得多——物体、布局、光照、指令全是新的。π0.5 的配方有几条主线，本节聚焦其中关于表示与知识保护的部分。

支撑泛化的底座是 co-training：把异构数据混在一起训——多本体数据、约 400 小时真实家庭移动操作、实验室数据、高层语义子任务预测、物体检测、以及大量 web 数据。web 数据是世界知识的来源（认得出「洗洁精」「微波炉」是什么、放哪），这也正是后面 knowledge insulation 要拼命保护的东西。

分层推理：先想「做什么」，再算「怎么动」

开放世界任务往往很长（先收拾台面 → 再洗碗 → 再擦灶台），直接端到端出低层动作很难。π0.5 用分层推理，让同一个模型内部含两条解码路径：

  • 高层：用离散自回归解码出一个语义子任务，比如 "pick up the plate"——这是文本，正是 VLM 擅长的离散 next-token 生成。
  • 低层：以这个子任务为条件，用连续 flow matching（5.2 那套）解出一段低层 action chunk，真正驱动关节。

这是一种「说出来再做出来」的结构：先用语言把意图想清楚（借力 web 知识做常识推理），再把意图翻译成连续控制。两条路径共用一个骨干，所以高层的语义理解能直接调制低层的动作生成。

从 RL 看 这是 options / 分层策略的味道

分层推理对应你在 RL 里见过的 hierarchical RL / options framework：一个高层策略选择「子目标 / 宏动作」（这里是语义子任务），一个低层策略在子目标条件下产出原始动作。区别在于：π0.5 不用 reward 训这个层级，而是全程模仿——高层子任务标注来自 co-training 数据里的语义标签，低层来自 flow matching。层级的好处一样：缩短低层的有效 horizon、让高层用语义抽象来泛化到没见过的场景。

两种表示的分工：FAST 离散预训练 vs flow matching 后训练

现在到核心张力：动作到底用离散还是连续表示？π0.5 的回答是——训练用离散、部署用连续，分两阶段。

  • 预训练阶段：把连续动作用 FAST action tokenizer 离散化成 token，然后用交叉熵 next-token 监督。这让 VLM 可以用它处理语言的那一整套机制来学控制表示——稳、快、易迁移，动作和文本在同一个离散 token 空间里，知识迁移最顺。
  • 后训练阶段：切换到 flow matching，做实时连续控制。离散 token 自回归一个一个吐太慢、精度受分箱限制，不适合高频实时；flow matching 一次积出整段、连续精细，适合部署。
离散（FAST + 交叉熵）连续（flow matching）
擅长训练稳定、知识迁移、复用语言机制实时、高精度、连续多模态
不擅长自回归慢、分箱损精度从零训随机初始化、易冲击骨干
π0.5 用在预训练 / 监督 backbone后训练 / 实时部署

一句话记忆：离散适合「学」和「迁移」，连续适合「跑」。

Knowledge insulation：用 stop-gradient 守住 web 知识

这是本节最精妙的一招（arXiv 2505.23705）。问题是这样的：你想给预训练好的 VLM 接一个随机初始化的 action expert，端到端一起训。但实验发现——让那个随机初始化的动作头的梯度直接回流（backprop）到 backbone，会损害训练速度、并破坏 VLM 学到的 web 知识迁移。

从 GAMES103 看 把一团噪声接到精心调好的系统上

想象你有一个已经收敛、状态良好的物理系统（预训练 VLM），现在硬接上一个初始全是噪声的新模块（随机 action expert），还让这个噪声模块的「力」直接反向作用回主系统。早期那些巨大、方向混乱的梯度，就像一股乱流冲击稳定结构，把原本调好的权重晃乱了——这正是 web 知识被冲掉、训练变慢的来源。解决思路也很物理：把这股乱流挡在门外。

Knowledge insulation（知识隔离）就是这道门，由两部分组成：

  • stop-gradient：照常训练 flow-matching action expert，但切断它回流到 backbone 的梯度。action expert 可以读 backbone 的表示（前向照常），但它的误差不许反向去改 backbone。backbone 不被随机噪声污染。
  • 离散 next-token 监督 backbone：那 backbone 靠什么学控制相关的东西？靠离散（FAST）next-token 这条干净、稳定的监督信号去微调——用语言那套机制更新 backbone，既稳又保住知识。

合起来效果是：backbone 由可靠的离散监督温和更新、保住 web 知识；action expert 在隔离层后专心学连续实时控制。web 知识保住了、训练更快了、推理仍然快（部署时还是 flow matching 那 10 步）。

openpi 现状

需要留意：开源的 openpi 目前只支持 flow matching head。也就是说，knowledge insulation 论文里那套「离散 FAST 监督 backbone + stop-gradient 隔离连续头」的完整双路训练，并非开箱即用。我们在 6.1 拆 EBench 的 openpi baseline 时，看到的是 flow-matching 这一路。

结果：进 104 个全新场景

这套配方换来的泛化能力很具体。π0.5 能走进全新的家，清理厨房、卧室，铺床，把毛巾挂上挂钩——这些都是多阶段、长流程任务，单次执行常达 10–15 分钟。论文报告：训练场景数量从约 3 个扩展到 104 个时，泛化表现持续改善，且大约 ~100 个场景后趋于饱和——也就是说，open-world 泛化对场景多样性的需求是有限而可达的，不是无底洞。

这一节你需要带走的 · 也是整个 Part 5 的收束

本节四点：(1) π0.5 在 π0 上加 co-training + 分层推理，冲 open-world；(2) 分层 = 离散自回归出高层语义子任务 + flow matching 出低层 chunk，同模型双路径；(3) FAST 让预训练把动作离散化、交叉熵 next-token，离散擅长学/迁移、连续擅长实时；(4) knowledge insulation = stop-gradient 切断随机 action expert 对 backbone 的梯度污染 + 用离散监督温和更新 backbone，保住 web 知识、训得更快。

至此，VLA 的「发动机」从 action chunking → flow matching → action expert → 离散/连续与知识隔离，已经完整拆开。你现在手里有了读懂任何一个现代 VLA 的全部理论零件。但零件不等于整机——一条训练样本长什么样？预训练、后训练、微调各自喂什么、更新谁？knowledge insulation 的 stop-gradient 到底剪在哪根线上？下一 Part（5.5），我们把这些零件装回一条真实的训练流水线，用 π0.5 和一个实验室特化的 LabVLA 做贯穿案例，补上「VLA 是怎么被造出来的」这张整机图纸。
