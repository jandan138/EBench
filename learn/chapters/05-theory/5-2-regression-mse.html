回归与 MSE：最基础的「拟合」

接下来的五节，我们要一步步走到 flow matching。但我答应你从零起步——所以这一节先把最朴素的一块地基打牢：什么叫「回归」，什么叫 MSE。后面每一种方法（diffusion、flow matching、rectified flow），本质都只是「回归一个不同的东西」。先把"回归"这个词彻底搞懂，后面就顺了。

先看一个最小的例子：拟合一条线

假设你收集了一堆数据点：每个点是一对数字 $(x, y)$。比如 $x$ 是"踩油门的力度"，$y$ 是"车速"。你把它们画在坐标纸上，发现大致排成一条向上的趋势，但有抖动、不完全在一条直线上。

现在你想要一个函数 $f$，给它一个新的 $x$，它能预测对应的 $y$。最简单的选择是一条直线 $f(x) = w\,x + b$，你要做的就是调 $w$（斜率）和 $b$（截距），让这条线尽量穿过这堆点。

这件事——用一个函数去拟合"输入 → 输出"的数据，使预测值尽量接近真实值——就叫 regression（回归）。它是监督学习里最基本的一类任务。"回归"这个名字是历史遗留，你完全可以在脑子里把它替换成"拟合"或"预测一个连续数值"，意思一样。

回归 vs 分类

如果输出 $y$ 是一个连续数值（车速 80.3 km/h、关节角 1.27 弧度），叫 regression。如果输出是从几个类别里选一个（猫/狗、左转/右转），叫 classification（分类）。机器人动作是连续的物理量，所以我们主要和回归打交道——这一点记住，到 5.8 讲"离散 token"时会形成对照。

怎么衡量"接近"？这就是 loss

"让线尽量穿过点"是句空话——计算机需要一个能算出数字的标准，告诉它"现在差多少"。这个标准叫 loss function（损失函数）：它吃进"预测值"和"真实值"，吐出一个数，越小代表拟合得越好。训练，就是不停调参数让这个数变小。

对每一个数据点，最自然的"差多少"就是预测值和真实值的差：$f(x_i) - y_i$。但这个差有正有负（预测高了为正，低了为负），直接加起来会正负抵消。于是我们把它平方（负号没了，而且差得越远惩罚越重），再对所有点取平均：

$$\text{MSE} \;=\; \frac{1}{N}\sum_{i=1}^{N}\bigl(f(x_i) - y_i\bigr)^2$$

这就是大名鼎鼎的 MSE（Mean Squared Error，均方误差）。逐字拆开：

  • Error（误差） = $f(x_i) - y_i$，预测减真实，一个点差多少。
  • Squared（平方） = $(\cdots)^2$，去掉正负号，并放大大误差。
  • Mean（均值） = $\frac{1}{N}\sum$，对 $N$ 个点求平均，得到一个总分。

MSE = 0 意味着每个预测都精确命中。MSE 越大，整体偏得越离谱。训练算法（梯度下降）就是顺着"让 MSE 变小"的方向，一点点挪动 $w, b$。

从 RL 看 你早就见过它

如果你做过 RL，MSE 对你不陌生：拟合 value function $V_\phi(s)$ 时，你最小化的 $\bigl(V_\phi(s) - \text{target}\bigr)^2$ 就是 MSE；critic 的回归、TD 误差的平方，全是这套。区别只在"回归的目标"不同——这恰恰是本系列的主线：每种方法都在回归某个东西，换的只是那个"东西"。

一个将决定一切的性质：MSE 偏爱"平均值"

这一节最重要的一句话，请慢读：当同一个输入 $x$ 对应多个不同的"正确" $y$ 时，最小化 MSE 会让模型输出它们的平均值。

为什么？想象同一个 $x$ 在数据里出现了两次，一次标注 $y=+1$，一次标注 $y=-1$（两个都对，只是两种做法）。模型只能为这个 $x$ 输出一个预测值 $\hat{y}$。它要让 $(\hat{y}-1)^2 + (\hat{y}+1)^2$ 最小——你求个导就知道，答案是 $\hat{y}=0$，正好是两者的平均。

可问题是：$+1$ 对、$-1$ 对，但平均出来的 $0$ 可能完全不对。这个看似无害的数学性质，正是传统 behavior cloning 在机器人上栽跟头的根源。下一节我们就用一个"绕开障碍物"的例子，把这个坑挖给你看。

这一节你需要带走的

三句话：(1) 回归 = 用函数拟合"输入→输出"、预测一个连续值；(2) MSE = 预测与真实之差，平方再平均，是衡量拟合好坏、驱动训练的标尺；(3) MSE 有个致命脾气——面对"多个都对的答案"，它会取平均。记住第三点，下一节见。
